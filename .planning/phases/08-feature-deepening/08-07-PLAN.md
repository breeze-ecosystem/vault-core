    1. **getSiteHealthScore(orgId: string)** — New method (D-15):
    - Fetch all camera health and reader health data for the org's sites
    - Call existing getCameraHealth() and getReaderHealth() which are already org-scoped
    - For each device, compute a 0-100 score:
      - Camera score: frame drop rate (inverse: 100 - drop%), heartbeat recency (0 if >5min ago), connection uptime ratio → weighted average
      - Reader score: response time (inverse: max(0, 100 - time/10)), auth failure rate (inverse: 100 - failure%) → weighted average
    - Aggregate: (total score across all devices) / (total device count) → round to integer
    - Return: { score, totalDevices, healthy (>=70), degraded (40-69), critical (<40) }
    - Follow pattern from PATTERNS.md lines 409-433

    2. **getSiteHealthScores(orgId: string)** — Grouped by site (D-15):
    - Same scoring but grouped by site/zone
    - Return: { siteId, siteName, score, totalDevices, healthy, degraded, critical, trend }[]

    3. **getDeviceHealth(orgId: string, deviceType?: 'CAMERA' | 'READER')** — New method (D-14):
    - Fetch device health data filtered by type
    - Include per-device metrics, status, and computed score
    - Return: { deviceId, deviceName, type, status, score, metrics: { ... }, lastHeartbeat, thresholdConfig }

    4. **Modify checkCameraHealth()** — Add frame drop rate tracking (D-14):
    - In the existing health check, add frame drop rate calculation
    - Compare against threshold from getHealthThresholds()
    - If frame drop rate exceeds threshold, emit alert (follow existing alert pattern)
    - Store frame drop data in the camera_health table

    5. **Modify checkReaderHealth()** — Add response time + auth failure tracking (D-14):
    - Add response time monitoring and auth failure rate tracking
