# Phase 8 — UI Design Contract

> Visual and interaction contract for Feature Deepening phase. Generated by gsd-ui-researcher.
> Covers **FTR-01 through FTR-07**: access control credential lifecycle, door state machine configurable thresholds, visitor host approval workflow, incident SLA profiles + evidence auto-bundle, ANPR confidence/thresholds, security analytics dashboard, equipment health dashboard, and mobile incident response + door control flows.
>
> **Builds on Phase 6 design system.** No new design system, no new npm packages. Extends and applies the existing Radix Themes + Tailwind + `motion` + `@repo/design` tokens.

## Spacing Scale

**Identical to Phase 6.** 8-point scale, already codified in `@repo/design/src/spacing.ts`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, compact cell padding |
| sm | 8px | Compact element spacing, badge padding, button icon gaps |
| md | 12px | Form field gaps, card-internal padding, config panel padding |
| base | 16px | Default spacing, section padding, card padding |
| lg | 24px | Section padding, dialog content, card groups |
| xl | 32px | Layout gaps between sections, page margins |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level padding, hero spacing |

**Exceptions** (unchanged from Phase 6):
- Mobile touch targets: minimum **44px** for icon-only buttons, quick-action tiles, tab bar items
- Header height: **56px** (`h-14`)
- Sidebar collapsed width: **64px**
- Sidebar expanded width: **240px**

**Source**: Phase 6 UI-SPEC (confirmed). No new spacing tokens needed. Configuration panels and new pages use existing scale.

## Color

**Identical to Phase 6.** Already codified in `@repo/design/src/colors.ts` and `globals.css`. HSL tokens unchanged.

### Dark Theme (default)

| Role | HSL Value | Usage in Phase 8 |
|------|-----------|-----------------|
| Dominant (60%) | `hsl(228 20% 4%)` — navy-black | Page backgrounds, main surfaces for all deepened pages |
| Secondary (30%) | `hsl(228 24% 6%)` / `hsl(228 16% 12%)` | GlassCard backgrounds, config panels, side panels, credential tables |
| Accent (10%) | `hsl(190 90% 50%)` — cyan #06b6d4 | **Reserved for**: primary CTAs, focus rings, active nav, severity-high indicators, online status dots, toggle active states, link text, slider active fill, selection highlight. **Never** decorative-only elements. |
| Destructive | `hsl(0 84% 60%)` — red #ef4444 | Revoke credential, cancel visit, deny vehicle, emergency lockdown, door forced-open alert |
| Warning | `hsl(35 92% 50%)` — amber #f59e0b | Degraded health status, expiring credentials, SLA breach warnings, borderline confidence scores |
| Success | `hsl(160 84% 39%)` — emerald #10b981 | Healthy equipment, approved visitor, allowed vehicle, door locked-secure, online status |
| Muted foreground | `hsl(228 10% 56%)` — slate #8892a6 | Secondary text, health score subtitles, empty state text, disabled config options |

### Light Theme

| Role | HSL Value | Usage |
|------|-----------|-------|
| Dominant | `hsl(0 0% 100%)` | Same layout as dark, inverted backgrounds |
| Accent | `hsl(190 90% 40%)` — deeper cyan #0a9abb | Same reservation list |
| All others | Complement of dark values from Phase 6 | — |

### Accent Reservation List (unchanged from Phase 6)

Cyan reserved for: primary button backgrounds (`variant="default"`), focus-visible rings, link text (`text-primary`), active nav indicator, selection highlight, alert severity high/critical indicators, status dot glow (online pulse), badge accent tint (`bg-primary/10`), toggle switch active, slider active fill, scrollbar hover thumb.

**Never use accent for**: decorative-only elements, non-interactive text, background fills, inactive states, borders of non-interactive cards, placeholder text.

### Semantic Color Extensions for Phase 8

New semantic mappings (same HSL values, new named contexts):

| Token | Used For |
|-------|----------|
| `health-critical` | red — health score 0-40%, SLA breach indicator |
| `health-degraded` | amber — health score 41-69%, borderline confidence |
| `health-healthy` | emerald — health score 70-100% |
| `credential-expired` | red — past expiry date |
| `credential-expiring` | amber — within 7 days of expiry |
| `credential-active` | emerald — valid and active |
| `anpr-confident` | emerald — score ≥ threshold |
| `anpr-uncertain` | amber — score below threshold but logged |
| `sla-pending` | amber — SLA timer running, within limits |
| `sla-breach` | red — SLA exceeded |

**Source**: Phase 6 UI-SPEC confirmed. Phase 8 adds semantic mappings only — no new HSL values.
