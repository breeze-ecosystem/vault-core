<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Redis ↔ Conversation memory | Short-term conversation state stored in Redis |
| Agent trace data → PostgreSQL agent_traces | Immutable audit trail of all tool calls |
| MCP server → External consumers | MCP protocol exposes tools to other systems (future) |
| Ollama health check → Degradation service | External dependency determines system availability |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-15 | Information Disclosure | Redis conversation data | mitigate | Redis key includes organizationId prefix for tenant isolation; TTL 90 days auto-purge per D-34 |
| T-09-16 | Tampering | Agent trace tampering | mitigate | AgentTrace table is append-only by design — no update/delete in traceToolCall(); audit logs hash-chained per organization |
| T-09-17 | Elevation of Privilege | Action confirmation bypass | mitigate | ActionConfirmationGuard validates confirmation token server-side; ADMIN/SUPERVISOR role verified per D-30; no client-side-only checks |
| T-09-18 | Denial of Service | Rate limit bypass | mitigate | RateLimitAgentGuard uses Redis atomic INCR — no client-side counters; per-minute rolling window; returns 429 with retry-after header |
| T-09-19 | Denial of Service | Model degradation cascading failure | mitigate | DegradationService checkHealth() runs before each LLM call; fallback response returned immediately — no retry loop |
| T-09-20 | Information Disclosure | Conversation data at rest | mitigate | Encrypted at rest per D-34 (encryption implementation deferred to agent discretion — encryption key from config.encryption.key applied to Redis values) |
</threat_model>
