## Color

| Role | Token | HSL | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--shadcn-background` | 228 20% 4% | Page background, main surface area |
| Secondary (30%) | `--shadcn-card` | 228 24% 6% | Cards, panels, sidebar, nav, chat bubbles (assistant) |
| Accent (10%) | `--shadcn-primary` | 190 90% 50% | **Reserved for**: primary CTA buttons, active nav indicators, agent response indicators, focus rings, risk gauge needle, SSE streaming cursor, accent glass panels |
| Destructive | `--shadcn-destructive` | 0 84% 60% | Destructive action buttons (lockdown confirm, revoke access), critical risk scores (85-100), door forced alerts |
| Success | `--shadcn-success` | 160 84% 39% | Online status indicators, low risk scores (0-33), pattern status "normal" |
| Warning | `--shadcn-warning` | 35 92% 50% | Medium risk scores (34-66), agent thinking indicator, pattern detection alerts |
| Muted FG | `--shadcn-muted-foreground` | 228 10% 56% | Secondary text, timestamps, agent status labels, empty state descriptions |
| Border | `--shadcn-border` | 228 16% 16% | Panel borders, input borders, chat message separators |

**Accent explicitly reserved for:** Primary CTA buttons ("Envoyer", "Expliquer", "Voir détails"), active navigation tab underline, agent message avatar ring, risk gauge active segment, SSE streaming text cursor animation, selected pattern card border, focus rings on interactive elements.

**NOT accent:** Body text, headings (use foreground), secondary buttons (use secondary/outline), info badges (use muted), success/warning indicators (use their semantic colors).

**Risk gauge color scale (FTR-10):**
- 0–33: `--shadcn-success` (green — normal)
- 34–66: `--shadcn-warning` (amber — elevated)
- 67–84: `--shadcn-primary` (cyan — high)
- 85–100: `--shadcn-destructive` (red — critical)

- `.glass` — default panel: card/60% + blur(12px) + border/50%
- `.glass-premium` — elevated panels: card/50% + blur(16px) saturate(1.2) + glow shadow
- `.glass-accent` — agent response panels: primary/4% + blur(16px) saturate(1.2) + primary/12% border

## Interaction Patterns

### SSE Streaming (Command Center + Mobile + Risk Explanation)

**Protocol:** Server-Sent Events via `EventSource` API (dashboard) / custom EventSource polyfill (React Native).

**Event types** (from RESEARCH.md Pattern 3):

| Event Type | Payload | UI Behavior |
|------------|---------|-------------|
| `thinking` | `{ agent: string, message: string }` | Show `AgentThinkingIndicator` with agent name |
| `token` | `{ content: string }` | Append token to `StreamingText` component with 50ms stagger |
| `tool_call` | `{ tool: string, args: object }` | Show collapsible tool call card in chat |
| `tool_result` | `{ tool: string, summary: string }` | Update tool call card with result summary |
| `agent_switch` | `{ from: string, to: string }` | Update `AgentStatusBar` active agent indicator |
| `error` | `{ message: string, recoverable: boolean }` | Show error inline or retry button per `recoverable` flag |
| `done` | `{ summary?: string }` | Close SSE stream, finalize message |

**Reconnection:** Exponential backoff (1s, 2s, 4s, 8s, max 30s). Show `SSEStatusBanner` during reconnection. Per RESEARCH.md Pitfall 4.

**Heartbeat:** Every 15s to detect stale connections. Auto-close after 5 minutes of inactivity. Max 1 concurrent SSE connection per user.

### Destructive Action Confirmation (D-30)

**Flow:**
1. Agent proposes destructive action in chat (e.g., "Je suggère de verrouiller la Zone B")
2. Action card renders in chat with: action name, target, impact description, `ConfirmationDialog` trigger button
3. Operator clicks "Exécuter" → `ConfirmationDialog` opens with:
   - Warning icon (AlertTriangle, amber)
   - Action description (from Copywriting Contract)
   - Confirm button (destructive variant, labeled with action verb)
   - Cancel button (outline variant)
4. On confirm: action executes, audit log entry created, chat shows result
5. On cancel: action dismissed, chat shows "Action annulée"

**RBAC enforcement:** ADMIN/SUPERVISOR role required. Non-authorized users see "Action non autorisée pour votre rôle." instead of confirm button.

### Risk Explanation Interaction (D-21)

**Flow:**
1. On `/risque` page, each zone risk score has an "Expliquer ce score" button
2. Click → `RiskExplanationPanel` slides open below the zone card
3. SSE stream starts: RiskAnalysisAgent analyzes the zone's recent events
4. Panel shows: streaming explanation text, color-coded gauge re-rendering, list of contributing events
5. Close button or click outside dismisses the panel

### Pattern Detection Interaction (D-22)

**Flow:**
1. `/patterns` page renders 8 `PatternCard` components in a responsive grid
2. Each card shows: pattern name, sparkline trend (7-day), current occurrence count, severity badge, "Voir les détails" button
3. Click "Voir les détails" → navigates to `/patterns/[id]` or expands inline
4. Detail view: full sparkline (30-day), occurrence table (paginated), "Expliquer" integration (PatternDetectionAgent trend analysis)

### Mobile Chat Interaction (D-23)

**Flow:**
1. More tab → Chat screen
2. Full-screen chat interface: message list (top), `QuickActions` bar (middle), input area (bottom)
3. **Voice input:** Tap mic button → recording state with waveform animation → Faster-Whisper transcription → auto-send transcribed text
4. **Text input:** Type message → tap send → SSE stream as per Command Center
5. **Quick actions:** Tappable chips for common queries — tap auto-sends the query
6. **Offline:** Grey out input, show offline banner. Actions like "Signalement" (incident capture) work offline and queue.
