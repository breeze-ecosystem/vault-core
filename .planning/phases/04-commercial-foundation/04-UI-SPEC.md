## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between avatar and label |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section padding, dialog/sheet content padding |
| xl | 32px | Layout gaps, card padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none

**Source**: Existing codebase — all components use Tailwind CSS default spacing scale (4px base). Header uses `h-14` (56px), buttons use `h-9` (36px), avatar uses `h-7` (28px), gap utilities use `gap-2` (8px) and `gap-3` (12px). This phase introduces no spacing changes.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 600 (semibold) | 1.2 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 24px | 600 (semibold) | 1.2 |

**Font Family**: IBM Plex Sans for all UI text (body, labels, headings). JetBrains Mono reserved for stat values and code (`font-mono tabular-nums`).

**Source**: Existing CSS tokens in `globals.css` and Tailwind config. `.stat-value` uses 24px font-mono semibold. `.section-title` uses 12px semibold uppercase. Body text uses `text-sm` (14px) throughout. No typography changes in this phase.

**Weight restriction**: Only 400 (regular) and 600 (semibold) are used. IBM Plex Sans is loaded at weights 300, 400, 500, 600, 700 but the design contract standardizes on just regular + semibold. Labels and headings use semibold; body uses regular.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | hsl(228 20% 4%) — navy-black | Page background, main surfaces |
| Secondary (30%) | hsl(228 16% 12%) / hsl(228 24% 6%) — dark navy | Cards, sidebar, nav, dropdown menus, dialogs, sheets |
| Accent (10%) | hsl(190 90% 50%) — cyan | **Reserved for**: primary button backgrounds, focus rings, active state indicators, status dot glow, badge background tint, link color, selection highlight |
| Destructive | hsl(0 84% 60%) — red | Destructive action buttons, revoke confirmation, expired invite status |

**Accent reserved for**: primary buttons (`variant="default"`), focus-visible rings (`--ring`), link text (`text-primary`), badge accent tint (`bg-primary/10`), selection background, active nav item indicator. **Never** used for: background fill, decorative-only elements, non-interactive text.

**Semantic color tokens** (defined in `globals.css` and mapped in `tailwind.config.ts`):
- `--warning`: hsl(35 92% 50%) — amber. Used for warning badges, expired invite status.
- `--success`: hsl(160 84% 39%) — emerald. Used for system status dot, active invite status.
- `--muted-foreground`: hsl(228 10% 56%) — slate. Used for secondary text, empty states, disabled text.

**Source**: Existing `globals.css` CSS custom properties. No color changes in this phase. The dark-only theme (no light mode — deferred to Phase 6) is enforced via `<html className="dark">`.

## Component / Interaction Behaviors

### 1. Organization Switcher (Dashboard Header Dropdown)

**Location**: Inside the existing avatar dropdown menu in `header.tsx`, between "Paramètres" and "Déconnexion".

**Behavior**:
- **Single organization**: Switcher is not rendered (no dropdown section). User sees only current org context in the header avatar area (display org name below user name instead of current role).
- **Multiple organizations**: Dropdown shows a "Changer d'organisation" section with current org marked "(actuelle)" and other orgs as clickable items. Clicking another org calls `POST /api/auth/switch-org`, re-issues tokens, updates AuthContext, and redirects to `/dashboard`.
- **Loading state**: The clicked org item shows a subtle spinner (Lucide `Loader2` with `animate-spin`) while the switch request is in flight.
- **Error state**: Toast notification with error message. User remains on current page (no redirect).

**Visual contract**:
- Uses existing `DropdownMenuLabel`, `DropdownMenuSeparator`, `DropdownMenuItem` components.
- Org items show: org name (bold), role badge (using existing Badge component, `variant="outline"`), "(actuelle)" suffix for active org.
- Icons: `Building2` (Lucide) for organization items.
- Height per org item: standard `DropdownMenuItem` height (~36px).

### 2. Organization Switcher (Mobile Bottom Sheet)

**Behavior**:
- Triggered from a `Building2` icon in the mobile header or profile screen.
- Opens a `Sheet` (side="bottom") listing all organizations.
- Current org has a checkmark and "(actuelle)" badge.
- Tapping another org executes switch (same API as dashboard), then navigates to home screen.
- Single org: no trigger rendered.

**Visual contract**:
- Sheet height: `max-h-[50vh]` with scrollable content.
- Sheet title: "Organisations" in `SheetTitle`.
- Org items: 48px touch target height. Show org name + role badge + member count.
- Icons: `Building2`, `Check` for current org.

### 3. Header Avatar Area Update

**Behavior change**: The user dropdown currently shows `user.role` below the name. After Phase 4:
- Replace `user.role` display with current organization name (e.g., "Acme Security").
- User's role is now per-organization — shown in the org switcher items, not in the header.

**Visual contract**:
- Existing layout unchanged: avatar (28px) + name (12px medium) + org name (10px muted) on desktop, hidden on mobile.
- Font: `text-xs` for name, `text-[10px]` for org name.

### 4. Invite Management Page (Dashboard)

**Route**: TBD by planner (likely `/parametres/invitations` or `/organisation/invites`).

**Layout**:
- Page header: "Invitations" with "Inviter un membre" button (primary, with `UserPlus` icon).
- Table of invites: Email, Rôle, Statut, Envoyée le, Actions columns (using existing Table component).
- Status badges: `En attente` (warning variant), `Acceptée` (success variant), `Expirée` (destructive variant), `Révoquée` (muted variant).
- Actions per row: `Renvoyer` (ghost button, `Send` icon) and `Révoquer l'invitation` (ghost button, destructive color, `X` icon).

**Create invite flow**:
- Button opens a Dialog (using existing Dialog component).
- Form fields: Email (Input with type="email"), Rôle (Select with options: Administrateur, Superviseur, Opérateur, Observateur).
- Form validation: email required + valid format, role required.
- Submit: POST to API, show success toast, close dialog, refresh table.
- Loading: Button shows spinner and "Envoi en cours..." during submission.

**Revoke flow**:
- Click "Révoquer" opens confirmation Dialog.
- Confirm calls DELETE API, shows toast, removes row from table.

### 5. Invite Accept Page (`/invite/[token]`)

**Route**: `apps/dashboard/app/invite/[token]/page.tsx` (Next.js App Router).

**States**:

**Loading state**:
- Full-page centered card with skeleton/spinner.
- Text: "Vérification de l'invitation..."

**Valid invite (new user)**:
- Card with: org name heading, role subtitle, form fields (firstName, lastName, password, confirmPassword), submit button.
- Form validation: all fields required, password min 8 chars, passwords must match.
- Submit: POST to accept API, on success redirect to `/dashboard` with new tokens.

**Valid invite (existing user)**:
- Card with: org name heading, role subtitle, "Rejoindre" button only (no password form).
- Submit: POST to accept API, on success redirect to `/dashboard`.

**Expired invite**:
- Card with error state: warning icon, "Cette invitation a expiré" heading, body text, link back to login.

**Already accepted**:
- Card with: "Invitation déjà acceptée" heading, body text, "Se connecter" button linking to `/login`.

**Invalid token**:
- Card with: "Invitation invalide" heading, body text.

### 6. Registration Form Update

**Location**: Existing registration page (TBD by planner — likely `/register` or `/auth/register`).

**Change**: Add one field: "Nom de l'organisation" (required text input) above or below existing email/password fields.

**Visual contract**:
- Uses existing `Input` and `Label` components.
- Group visually with a subtle separator or heading: "Informations de l'organisation".
- Validation: required, min 2 characters, max 100 characters.

### 7. Auth Context Extension

**State additions to AuthContext**:
```typescript
interface AuthContextType {
  user: User | null;
  organization: { id: string; name: string } | null;  // NEW
  organizations: Array<{ id: string; name: string; role: string }>;  // NEW
  switchOrganization: (orgId: string) => Promise<void>;  // NEW
  isLoading: boolean;
  isAuthenticated: boolean;
  logout: () => Promise<void>;
}
```

**Fetch organizations**: On auth init, fetch `GET /api/auth/organizations` to populate the switcher list. Only fetched if user has > 1 org membership.
