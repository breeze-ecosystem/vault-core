| Boundary | Description |
|----------|-------------|
| Zod schema → API request | Schemas define validation rules at the boundary between untrusted input and application logic |
| Shared package → API consumers | Schema types are imported by API controllers; type mismatches cause runtime validation errors |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Tampering | registerSchema — organizationName | mitigate | z.string().min(1) prevents empty org name; no max length constraint per UI-SPEC (max 100 chars) — add if needed |
| T-04-07 | Elevation of Privilege | createInviteSchema — role field | mitigate | z.enum(["ADMIN", "SUPERVISOR", "OPERATOR", "VIEWER"]) restricts to valid roles. SUPER_ADMIN is excluded from invite creation — only assigned at DB level. |
| T-04-08 | Information Disclosure | switchOrgSchema — organizationId | mitigate | UUID validation prevents arbitrary string injection. Actual membership check is in AuthService (Plan 04), not in Zod schema. |
| T-04-09 | Tampering | acceptInviteSchema — token | mitigate | Token is JWT-signed and verified server-side (Plan 04). Zod min(1) just prevents empty string. |
</threat_model>
