          - If enabled → return true, else throw ForbiddenException

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request → FeatureGateGuard | Feature key from decorator metadata is trusted (compile-time constant). orgId from JWT is verified by JwtStrategy. |
| FeatureGateGuard → Redis | Cache is trusted; 5-min TTL limits stale-cache window. Cache key includes orgId — no cross-org cache leak. |
| FeatureGateGuard → FeatureFlag table | Query is auto-scoped by Prisma extension (FeatureFlag in SCOPED_MODELS). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-26 | Elevation of Privilege | Feature gate bypass via missing decorator | accept | Feature gates are additive — they restrict, not grant. Endpoints without @RequiresFeature are accessible to all orgs. Protecting endpoints is a developer responsibility. |
| T-04-27 | Elevation of Privilege | Redis cache poisoning — feature flag override | mitigate | Cache key includes orgId (`feature:{orgId}:{feature}`). Cross-org cache poisoning requires knowing another org's UUID. 5-min TTL limits window. |
| T-04-28 | Denial of Service | Redis unavailable — every request hits DB | accept | FeatureGateGuard falls back to DB query on Redis miss. DB load increase acceptable given existing Redis uptime. |
| T-04-29 | Elevation of Privilege | Direct FeatureFlag table modification | mitigate | FeatureFlag is in SCOPED_MODELS (Prisma extension scoping). Admin-only access via RolesGuard. Audit log records all FeatureFlag changes. |
</threat_model>
