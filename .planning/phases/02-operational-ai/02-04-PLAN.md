    **Part B — Python ANPR Endpoint (services/ai-preprocessor/main.py):**

    Add a new endpoint to the existing FastAPI application (append after existing route handlers):

    ```python
    from paddleocr import PaddleOCR
    import base64
    import io
    from PIL import Image
    import logging

    logger = logging.getLogger(__name__)

    # Lazy-load PaddleOCR on first request (cold start)
    _ocr = None
    def get_ocr():
        global _ocr
        if _ocr is None:
            logger.info("Initializing PaddleOCR (first call may be slow)...")
            _ocr = PaddleOCR(use_angle_cls=True, lang='en')
        return _ocr

    class ANPRRequest(BaseModel):
        image_base64: str
        camera_id: str

    class PlateResult(BaseModel):
        plate: str
        confidence: float
        bbox: list

    class ANPRResponse(BaseModel):
        plates: list[PlateResult]
        camera_id: str
        processing_time_ms: float

    @app.post("/api/v1/anpr", response_model=ANPRResponse)
    async def recognize_plate(request: ANPRRequest):
        """Detect and recognize license plate from camera frame using PaddleOCR."""
        import time
        start = time.time()

        # Decode base64 image
        try:
            image_data = base64.b64decode(request.image_base64)
            image = Image.open(io.BytesIO(image_data))
        except Exception as e:
            logger.error(f"Failed to decode image: {e}")
            raise HTTPException(status_code=400, detail="Invalid image data")

        # Run OCR
        ocr = get_ocr()
        results = ocr.ocr(image, cls=True)

        plates = []
        if results and results[0]:
            for word_info in results[0]:
                text = word_info[1][0]
                confidence = word_info[1][1]
                bbox = word_info[0]

                # Filter: only include results with reasonable confidence (>= 0.5)
                if confidence >= 0.5:
                    plates.append(PlateResult(
                        plate=text.strip().upper(),
                        confidence=round(float(confidence), 4),
                        bbox=[float(coord) for point in bbox for coord in point],
                    ))

        elapsed = round((time.time() - start) * 1000, 2)
        logger.info(f"ANPR processed {len(plates)} plates in {elapsed}ms for camera {request.camera_id}")

        return ANPRResponse(
            plates=plates,
            camera_id=request.camera_id,
            processing_time_ms=elapsed,
        )
    ```

    **Part A — ANPR Service (anpr.service.ts):**

    Create service following RESEARCH.md Pattern 5 (lines 687-741):

    1. `analyzePlate(frame: string, cameraId: string): Promise<PlateResult[]>`:
       - POST to AI Preprocessor: `${this.aiBaseUrl}/api/v1/anpr` with { image_base64: frame, camera_id: cameraId }
       - 15-second timeout via AbortSignal.timeout(15000)
       - Parse response, handle errors (timeout, network, invalid response)
       - Return list of PlateResult { plate, confidence, bbox }

    2. `evaluatePlate(plate: string, siteId: string): Promise<{ decision: VehicleDecision, reason: string }>`:
       - Check Redis cache first: key `vehicle:allowlist:{plate}` — if cached, return ALLOW
       - Check Redis cache: key `vehicle:blocklist:{plate}` — if cached, return DENY
       - If not cached, query Prisma VehicleList for the plate+siteId:
         - If allowlist entry exists, cache in Redis (TTL 1 hour), return ALLOW
         - If blocklist entry exists, cache in Redis (TTL 1 hour), return DENY
         - If neither, return { decision: "DENY", reason: "unknown" }
       - The Redis caching prevents Prisma queries on every frame for known plates

    3. `recordEvent(plateData: { plate, confidence, cameraId, siteId, imageUrl }, decision, reason)`:
       - Insert into vehicle_events hypertable via $queryRaw:
         ```typescript
         await this.prisma.$queryRaw`
           INSERT INTO vehicle_events (time, site_id, camera_id, plate, confidence, image_url, decision, reason)
           VALUES (NOW(), ${siteId}::uuid, ${cameraId}::uuid, ${plate}, ${confidence}, ${imageUrl}, ${decision}::vehicle_decision, ${reason})
         `;
         ```
       - If decision is DENY and reason is "blocklist", emit `equipment.alert` event for alert creation
       - If decision is ALLOW, emit event for potential MQTT gate open signal (deferred infrastructure, log intent)

    4. `processFrame(frame: string, cameraId: string, siteId: string)`:
       - Orchestrator: call analyzePlate → for each plate found, call evaluatePlate → call recordEvent
       - This is called from the BullMQ processor (not sync HTTP)

    5. **Allowlist/Blocklist CRUD:**
       - `createListEntry(dto)`: Create VehicleList record. After creation, flush Redis cache for this plate.
       - `updateListEntry(id, dto)`: Update VehicleList. Flush Redis cache.
       - `deleteListEntry(id)`: Delete VehicleList record. Flush Redis cache.
       - `listEntries(type, siteId)`: Query Prisma with type filter (allowlist/blocklist), site scope
       - `getEntry(id)`: Single entry

    6. `queryEvents(filters)`: Query vehicle_events hypertable via $queryRaw with filters:
       - plate (ILIKE search), siteId, from/to time range, decision type
       - Paginated with ORDER BY time DESC
       - Return { data, total, page, limit }

    7. **onModuleInit — Cache warm:**
       - On startup, load all active allowlist and blocklist entries into Redis cache
       - Key format: `vehicle:allowlist:{plate}` → "1", TTL 1 hour
       - This ensures plate evaluation doesn't always miss cache on first recognition after restart
