<objective>
Deliver AI natural language features as a complete vertical slice. By the end of this plan, operators can query the security system in natural language and receive matching events with contextual summaries, incidents automatically generate AI-powered closure summaries, and the AI assistant answers questions about building state, recent events, and zone status using semantic search over pgvector event embeddings.

Purpose: AI natural language interaction makes the platform accessible beyond security specialists. Instead of navigating complex filter forms, operators can ask "what happened after midnight on Site A?" and get instant, relevant results. AI incident summaries reduce post-incident report writing from hours to seconds.

Output: Natural language event query with Ollama LLM, auto-generated incident summaries, and RAG-style AI assistant with pgvector semantic search.
</objective>

    ```sql
    -- AI-01, AI-03: Event embeddings for semantic search
    -- Requires: CREATE EXTENSION IF NOT EXISTS vector;
    -- Run before: psql -f 012_event_embeddings.sql

    CREATE TABLE event_embeddings (
        id         UUID DEFAULT gen_random_uuid(),
        time       TIMESTAMPTZ NOT NULL,
        site_id    UUID NOT NULL,
        event_type VARCHAR(32) NOT NULL,
        event_id   UUID NOT NULL,
        summary    TEXT NOT NULL,
        embedding  VECTOR(768) NOT NULL  -- Ollama nomic-embed-text dimension
    );

    SELECT create_hypertable('event_embeddings', 'time',
        chunk_time_interval => INTERVAL '7 days'
    );

    CREATE INDEX idx_event_embeddings_site ON event_embeddings (site_id, time DESC);
    CREATE INDEX idx_event_embeddings_vector ON event_embeddings USING hnsw (embedding vector_cosine_ops)
        WITH (m = 16, ef_construction = 200);
    ```

         ```typescript
         const similarEvents = await this.prisma.$queryRawUnsafe<...>(

                   embedding <=> $1::vector AS similarity
            FROM event_embeddings
            WHERE time >= NOW() - INTERVAL '7 days'
            ORDER BY embedding <=> $1::vector
            LIMIT 5`,

         );
         ```
