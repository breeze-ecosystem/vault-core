<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client (Edge Agent) → Mosquitto:8883 | TLS-encrypted MQTT traffic over WAN |
| Mosquitto:1883 → localhost | Internal, no TLS, localhost-only |
| Edge Agent container → Host serial devices | Raw device access via /dev/tty* |
| Edge Agent container → Host network interface | Full host network access via --network=host |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-INF-01 | Tampering, Info Disclosure | MQTT broker exposed without auth | mitigate | `allow_anonymous false`; `password_file` with hashed credentials; listener 1883 bound to localhost only per mosquitto.conf |
| T-INF-02 | Spoofing, Tampering, Info Disclosure | MITM on MQTT traffic | mitigate | TLS listener on 8883 with `cafile`/`certfile`/`keyfile` configured; clients use `CERT_REQUIRED` to verify server cert against CA |
| T-INF-03 | Elevation of Privilege | Container breakout via serial device | mitigate | `--device` flags per port (not `--privileged`); each serial device explicitly mapped; no container-wide privileged mode |
| T-INF-04 | Spoofing | Event injection via spoofed MQTT publisher | mitigate | Per-site ACLs restrict each agent to its site topic namespace; `topic read site/{siteId}/door/+/cmd` / `topic write site/{siteId}/door/+/state` in ACL file |
| T-INF-05 | Info Disclosure | Credential leak in env vars | accept | Docker secrets mechanism not used in Phase 1; credentials passed via env vars; documented as temporary until Phase 2 enrollment API |
| T-INF-06 | Spoofing | Unauthenticated agent connecting to broker | mitigate | Username/password authentication required via `password_file`; `mosquitto_passwd` creates hashed entries |
| T-INF-SC | Tampering | Package installs (Alpine openssl, Mosquitto image) | mitigate | All images from official Docker Hub (`alpine:3`, `eclipse-mosquitto:2.0`); assumed legitimate. No [SUS] flagged packages. |
</threat_model>
