<objective>
Rewrite the Edge Agent from synchronous polling (`schedule` + `httpx`) to a native `asyncio` event loop architecture with concurrent serial I/O, MQTT pub/sub, and HTTP operations (INF-01). Preserve all existing monitoring capabilities (heartbeat, health checks, Docker management, PostgreSQL backups).

Purpose: Enable concurrent hardware protocol handling — the foundation for Phase 2 OSDP door control and ONVIF camera discovery. Without async, a blocking serial read would stall MQTT command processing and health reporting.

Output: Complete async Edge Agent codebase in `edge/agent/` with graceful lifecycle management, MQTT reconnect with bounded buffer, and ONVIF WS-Discovery probe service.
</objective>

  **1. `edge/agent/tasks/serial_task.py`**
  - `async def serial_reader(shutdown: asyncio.Event, device: str, baud: int, message_queue: asyncio.Queue) -> None`
  - Uses `serial_asyncio.open_serial_connection(url=device, baudrate=baud)` to get `reader, writer`
  - Loop reads data with `asyncio.wait_for(reader.readexactly(1), timeout=1.0)` — the 1s timeout prevents blocking starvation (per Pitfall 5 from RESEARCH.md)
  - On data received, reads the full OSDP frame (for now: collect bytes until idle; Phase 2 will add proper OSDP frame parsing)
  - Puts received data onto `message_queue` for the MQTT task to publish
  - On `asyncio.TimeoutError`: re-loop (expected, no action)
  - `try/finally` block closes `writer` and `await writer.wait_closed()` on shutdown (per Pitfall 6)
  - Logs connection, disconnection, and errors

  **2. `edge/agent/tasks/mqtt_task.py`**
  - `async def mqtt_handler(shutdown: asyncio.Event, settings) -> None`
  - Build `aiomqtt.TLSParameters` with `ca_certs=settings.MQTT_TLS_CA_CERT, certfile=None, keyfile=None, cert_reqs=ssl.CERT_REQUIRED` — per D-06 (no client certs)
  - Outer `while not shutdown.is_set()` reconnect loop with `reconnect_interval=settings.MQTT_RECONNECT_INTERVAL`
  - Inside the loop: `async with aiomqtt.Client(hostname=..., port=8883, username=..., password=..., tls_params=...) as client:`
  - On connect:
    1. Drain replay buffer: pop and publish all buffered messages
    2. Subscribe to `site/{site_id}/door/+/cmd` (door commands from API)
    3. Enter message loop with `async with client.messages() as messages:`
  - Message parsing: parse JSON payload, log, and forward to serial task via `message_queue`
  - Publish helper: `async def publish_state(client, topic_prefix, payload_model)`
  - `asyncio.Queue(maxsize=settings.MQTT_BUFFER_MAXSIZE)` for the buffer — per D-16 bounded queue
  - Buffer logic:
    - When client is disconnected (outside the `async with` block), all publish calls put messages on the queue
    - When queue is full, log warning and drop oldest (FIFO eviction via `queue.get_nowait()`)
    - On successful reconnect, drain the queue before resuming normal operation
  - Track per-device sequence numbers with `defaultdict(int)` for outgoing messages (follows the same sequence dedup pattern as MqttService in the API)
  - `except aiomqtt.MqttError`: log warning, sleep reconnect_interval, retry
  - Notify shutdown event on exit
