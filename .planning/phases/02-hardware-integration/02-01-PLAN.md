<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Edge Agent ↔ RS-485 bus | Physical serial bus carrying OSDP frames; physically secured but not encrypted without SC |
| Edge Agent ↔ MQTT broker | TLS-encrypted MQTT (Phase 1); all hardware traffic goes through authenticated channel |
| Edge Agent ↔ ONVIF camera | Local LAN HTTP SOAP; credentials sent in HTTP headers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | OSDP controller impersonation on RS-485 | mitigate | OSDP Secure Channel (AES-128) per-session key prevents replay (D-01); fallback to plain OSDP logged as warning |
| T-02-02 | Information Disclosure | ONVIF credentials in Edge Agent config | mitigate | ONVIF_USERNAME/PASSWORD passed via env vars (not hardcoded); Docker secrets support in future |
| T-02-03 | Tampering | OSDP command injection on RS-485 bus | mitigate | SC encryption prevents command injection on plaintext; fallback mode logs unencrypted commands for audit |
| T-02-04 | Denial of Service | OSDP refresh interval starvation | mitigate | Dedicated asyncio task at 50ms with priority; other tasks run in separate coroutines |
| T-02-SC | Tampering | npm/pip package installs | mitigate | All packages verified via PyPI JSON API (see RESEARCH.md Package Legitimacy Audit); `[ASSUMED]` disposition |
</threat_model>
