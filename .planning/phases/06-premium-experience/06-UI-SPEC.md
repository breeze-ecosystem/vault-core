### CSS Variable Architecture

**Current state** (`globals.css`): All design tokens defined as HSL CSS custom properties on `:root`. Tailwind `extend` maps `colors: { background: "hsl(var(--background))" }` etc. Dark theme only.

**Target state** (namespace migration per D-01, D-02):

1. Prefix existing shadcn/ui CSS vars with `--shadcn-`:
   - `--background` → `--shadcn-background`
   - `--primary` → `--shadcn-primary`
   - `--card` → `--shadcn-card`
   - All 17 existing CSS vars get `--shadcn-` prefix.

2. Add Radix Themes CSS import + provider wrapper (`Theme` from `@radix-ui/themes`).

3. Map Radix Themes CSS vars to Tailwind utility classes to maintain `bg-background`, `text-primary` etc. via Tailwind `extend`.

4. Add **light mode overrides** on `[data-theme="light"]` or `.light` selector:
   - `--shadcn-background`: lighter shade (e.g., `hsl(0 0% 100%)`)
   - `--shadcn-foreground`: darker shade (e.g., `hsl(228 20% 4%)`)
   - All tokens get a complementary light value.

5. **File migration ordering**: `globals.css` → `tailwind.config.ts` → `components/ui/*` (find+replace references to renamed vars) → Radix Themes provider in root layout → individual page components.

## Spacing Scale

Declared values (multiples of 4, 8-point scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, avatar-label spacing |
| sm | 8px | Compact element spacing, badge padding, button icon gaps |
| md | 12px | Form field gaps, card-internal padding |
| base | 16px | Default element spacing, section padding |
| lg | 24px | Section padding, card padding, dialog content padding |
| xl | 32px | Layout gaps between sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level padding, hero spacing |

**Exceptions**:
- Mobile touch targets: minimum **44px** (iOS HIG guideline) for icon-only buttons, quick-action tiles, tab bar items. This overrides the scale for those specific elements.
- Header height: **56px** (`h-14`) — kept from existing layout.
- Sidebar collapsed width: **64px** — kept from existing layout.
- Sidebar expanded width: **240px** — kept from existing layout.

**Source**: Existing codebase uses Tailwind CSS default spacing scale (4px base). Phase 6 introduces no changes to the fundamental scale. The glass panels and cards use `p-5` (20px) as standard internal padding.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px (text-3xl) | 600 (semibold) | 1.2 | Page hero titles, overview metric hero values |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | Section headings, card titles, page headings |
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Paragraph text, table cells, descriptions |
| Label | 12px (text-xs) | 600 (semibold) / 400 (regular) | 1.2 | Section labels, stat labels, badge text, uppercase section titles (600 weight). Timestamps, metadata, helper text (400 weight) |

**Monospace usage** (JetBrains Mono):
- `stat-value` class: 24px (text-2xl), 600 weight, tabular-nums — for large metric values
- Time/duration displays: 14px (text-sm), 600 weight, tabular-nums
- Code blocks, IDs, serial numbers: 13px (text-xs), 400 weight

**Weight restriction**: Only **400 (regular)** and **600 (semibold)** are used for UI text. IBM Plex Sans is loaded at weights 300–700 but 300/500/700 are reserved for:
- 300: Very large display text (optional, hero word-marks)
- 500: Not used in this phase
- 700: Not used in this phase (600 provides sufficient emphasis)

**Line height standardization**:
- Display/Hero: 1.1 — tight for large display numbers
- Headings: 1.2 — standard for heading tightness
- Labels: 1.2 — tight for compact UI elements
- Body text: 1.5 (21px at 14px) — ensures readability

**Source**: Extended from existing CSS tokens. The `.stat-value` class (24px font-mono semibold) and `.section-title` class (12px semibold uppercase) remain. Body text uses `text-sm` (14px) throughout. New 28px display size added for hero metrics. Label/small text uses 12px at 400 weight for captions and metadata.

## Animation Contract

### Library: `motion` (successor to framer-motion)

Install as new dependency: `npm install motion` (~15KB gzipped).

### Two-Layer Strategy (D-06)

#### Layer 1: Page/Route Transitions (AnimatePresence)

Applied in the dashboard layout wrapper (`dashboard-layout.tsx` or new `page-transition.tsx`):

| Transition | Duration | Easing | Effect |
|-----------|----------|--------|--------|
| Page enter | 300ms | `easeOut` (cubic-bezier(0.16, 1, 0.3, 1)) | Fade up + slight scale: `opacity: [0, 1]`, `y: [12, 0]` |
| Page exit | 200ms | `easeIn` | Fade down: `opacity: [1, 0]`, `y: [0, -8]` |
| Route change | 350ms | system | Content crossfade with sidebar/header remaining static |

#### Layer 2: Component-Level Micro-Interactions

| Component | Trigger | Effect | Duration | Easing |
|-----------|---------|--------|----------|--------|
| Stats cards (grid) | Page load | Staggered entrance from children: `y: [20, 0]`, `opacity: [0, 1]` | 400ms each, stagger 80ms | `easeOut` |
| CameraGrid cards | Page load / filter | Staggered scale entrance: `scale: [0.95, 1]`, `opacity: [0, 1]` | 300ms each, stagger 60ms | `easeOut` |
| AlertFeed rows | Page load | Sequential slide-in from right: `x: [20, 0]`, `opacity: [0, 1]` | 250ms each, stagger 40ms | `easeOut` |
| Side panel | Open | Slide from right: `x: [320, 0]` | 250ms | `easeOut` |
| Side panel | Close | Slide to right: `x: [0, 320]` | 200ms | `easeIn` |
| Hover — CameraCard | Mouse enter | Scale 1.02, border glow | 200ms | `easeOut` |
| Hover — stat card icon | Mouse enter | Rotate + scale icon to 1.1 | 150ms | `easeOut` |
| Notification toast | Show | Slide down from top: `y: [-60, 0]` | 300ms | spring (stiffness 300, damping 25) |
| Notification toast | Dismiss | Slide up: `y: [0, -60]` | 200ms | `easeIn` |
| Button hover | Mouse enter | Subtle brightness 1.05 | 150ms | `easeOut` |
| Bulk action bar | Appear | Slide up from bottom + fade | 250ms | `easeOut` |
| Alert severity dot | New alert | Pulse + glow animation (2 cycles) | 600ms | `easeInOut` |
| Quick-action toolbar buttons | Hover | Scale 1.05 + icon shift 2px | 150ms | `easeOut` |
| Theme toggle | Click | Rotate icon 180deg + background color transition | 300ms | `easeOut` |

### Motion Best Practices

- **`useReducedMotion()`**: Check `prefers-reduced-motion` and disable all non-essential animations. Page transitions remain as instant swaps (0ms duration).
- **`layoutId`**: Use shared layout animations for filter transitions (grid ↔ map view toggle on Cameras page).
- **`AnimatePresence` mode**: `"wait"` for page transitions, `"popLayout"` for filter changes within pages.
- **No `motion` on mobile** — mobile uses React Native's `Animated` API or `react-native-reanimated` (not yet installed) for any transitions. Phase 6 mobile uses CSS-transition-like built-in tab transitions only. Animation on mobile deferred to Phase 8+.

## Component & Interaction Specifications

### 1. Theme Provider & Dark/Light Mode Toggle

**Location**: 
- Provider: Root layout (`apps/dashboard/app/layout.tsx`), wraps children
- Toggle: **Sidebar bottom section** (near user avatar, before logout) + system preference detection

**Behavior**:
- On mount: read `localStorage.getItem("theme")`. If `null`, check `prefers-color-scheme: dark`. Apply corresponding data attribute.
- Toggle cycles: `system` → `dark` → `light` → `system` (actual UI: switch with 3 states managed via a dropdown or a toggle with auto/dark/light options)
- Persist choice to `localStorage.setItem("theme", value)`
- Set `data-theme` attribute on `<html>` element (allows CSS variable switching)
- System preference change listener: if current mode is `system`, re-evaluate on change

**Visual contract**:
- Toggle component: 44px touch target, shows sun icon (light) / moon icon (dark) / monitor icon (auto)
- Background transition: `background-color` 300ms ease-out on `<html>` (CSS transition on the background variable)
- All Radix Themes components re-render automatically via `Theme` provider's `appearance` prop

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| Keyboard navigation | All interactive elements reachable via Tab. Radix Themes primitives handle ARIA roles. Custom components use Radix primitives. |
| Focus indicators | `--ring` color (accent cyan) visible on all interactive elements. Minimum 2px offset. |
| Color contrast | Dark theme: Body text (14px) on background (hsl(228 20% 4%)) — contrast ratio > 13:1. Accent (#06b6d4) on dark backgrounds > 4.5:1 minimum. |
| Motion sensitivity | `prefers-reduced-motion` respected — all `motion` animations disabled. Page transitions become instant. Micro-interactions removed. |
| Screen reader | Radix Themes primitives include built-in ARIA attributes. Custom components use `aria-label`, `role`, `aria-expanded` as needed. |
| Theme toggle | Button has `aria-label="Passer en mode clair/sombre"`. System preference respected. |
| Semantic HTML | Proper heading hierarchy (h1 per page, h2 for sections). Landmark elements (`<nav>`, `<main>`, `<aside>`) used. |

## Performance Requirements

| Requirement | Target |
|-------------|--------|
| Animation frame rate | 60fps during transitions. Use `will-change: transform, opacity` on animated elements. GPU-composited properties only (`transform`, `opacity`). |
| Bundle impact | `motion` adds ~15KB gzipped. Radix Themes adds ~30KB gzipped. Total new JS: ~50KB gzipped. No regressions on TTI. |
| Image loading | Camera thumbnails use `loading="lazy"` for below-fold. Thumbnails in grid use `priority` for first 4 visible. |
| Animation timing | Staggered entrance animations complete within 600ms total. Page transitions complete within 350ms. No animation blocks interaction. |
| List virtualization | Alert feed: Virtualize if > 50 rows (use `react-window` if needed — not yet installed). Camera grid: paginate at 32 items with "Charger plus" button. |
| CSS transition vs `motion` | Prefer CSS transitions for simple state changes (hover, focus, color). Use `motion` for enter/exit, layout, and staggered animations. |
