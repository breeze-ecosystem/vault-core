<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| Mobile app → API (geofencing heartbeat) | Untrusted SSID/userId sent from mobile app |
| API → Hermes Agent | Alert data sent via HTTP webhook (internal network) |
| API → GSM modem | AT commands sent via serial port (local hardware) |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Spoofing | Geofencing heartbeat | mitigate | SSID validated against trustedSsids list from config; userId extracted from authenticated JWT, not from request body |
| T-02-08 | Tampering | Modem AT commands | mitigate | ModemService sanitizes message text to GSM 03.38 charset; prevents AT command injection by stripping null bytes and CR/LF from message body |
| T-02-09 | DoS | WhatsApp queue retry | mitigate | Maximum retry count per notification (max 5 retries with exponential backoff); rate limit to 1 msg/sec via Hermes |
| T-02-10 | Info Disclosure | Alert content in WhatsApp/SMS | accept | VISION tier alerts contain camera name and event type only; no PII. SMS over GSM is unencrypted by nature |
| T-02-SC | Tampering | npm packages | mitigate | HUMAN VERIFY @serialport/stream and @nestjs/schedule before npm install (ASSUMED per RESEARCH.md) |
</threat_model>
