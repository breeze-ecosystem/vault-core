    - `AuditLogEntry` interface (id, action, entityType, entityId, userId, userName, details, ipAddress, timestamp)
    - `fetchAuditLogs(params?): Promise<PaginatedResponse<AuditLogEntry>>` — GET /api/audit

    **API Keys:**
    - `ApiKey` interface (id, name, key, status, lastUsed, createdAt, expiresAt)
    - `fetchApiKeys(params?): Promise<PaginatedResponse<ApiKey>>` — GET /api/api-keys
    - `createApiKey(data): Promise<ApiKey>` — POST /api/api-keys
    - `revokeApiKey(id): Promise<void>` — DELETE /api/api-keys/:id

    **Compliance:**
    - `ComplianceReport` interface (id, title, status, generatedAt, period, metrics)
    - `fetchComplianceReports(params?): Promise<PaginatedResponse<ComplianceReport>>` — GET /api/compliance

    **Webhooks:**
    - `WebhookSubscription` interface (id, name, url, events, status, lastDelivery, createdAt)
    - `fetchWebhooks(params?): Promise<PaginatedResponse<WebhookSubscription>>` — GET /api/webhooks
    - `createWebhook(data): Promise<WebhookSubscription>` — POST /api/webhooks
    - `updateWebhook(id, data): Promise<WebhookSubscription>` — PATCH /api/webhooks/:id
    - `deleteWebhook(id): Promise<void>` — DELETE /api/webhooks/:id
    - `testWebhook(id): Promise<{success, statusCode}>` — POST /api/webhooks/:id/test

    **Schemas:**
    - `SchemaDefinition` interface (id, name, version, schema, isActive)
    - `fetchSchemas(params?): Promise<PaginatedResponse<SchemaDefinition>>` — GET /api/schemas

    **Equipment:**
    - `EquipmentItem` interface (id, name, type, status, siteId, siteName, lastHeartbeat, metrics)
    - `fetchEquipment(params?): Promise<PaginatedResponse<EquipmentItem>>` — GET /api/equipment

    **Maintenance:**
    - `MaintenanceTicket` interface (id, title, description, status, priority, assignedTo, createdAt, resolvedAt)
    - `fetchMaintenanceTickets(params?): Promise<PaginatedResponse<MaintenanceTicket>>` — GET /api/maintenance
    - `createMaintenanceTicket(data): Promise<MaintenanceTicket>` — POST /api/maintenance

    **Patterns:**
    - `SecurityPattern` interface (id, name, description, type, severity, detectedAt, status)
    - `fetchPatterns(params?): Promise<PaginatedResponse<SecurityPattern>>` — GET /api/patterns

    Follow these rules:
    1. Use the same `fetchWithAuth` approach as existing functions
    2. Error message format: `"Erreur lors du chargement des {items}"` (French, matches existing pattern)
    3. Type names follow existing conventions
    4. Use `PaginatedResponse<T>` from existing types
    5. Verify the actual API endpoint exists by checking `apps/api/src/modules/` for the matching controller
    6. If an exact endpoint doesn't match the Dashboard's client, use the Dashboard api.ts as reference for the correct URL pattern
