    contact.service.ts:
    - Inject PrismaService, ConfigService, NotificationService via constructor
    - async handleContact(data):
      1. Verify Turnstile: POST to https://challenges.cloudflare.com/turnstile/v0/siteverify with secret key from ConfigService + token
      2. If fails: throw BadRequestException("Verification failed. Please try again.")
      3. Save to DB: prisma.contactSubmission.create({ data: { name, email, company, message } })
      4. Send email via NotificationService (Resend) to configured sales notification address
      5. Return { success: true }
  </action>

|----------|-------------|
| Browser → API | Contact form data POSTs from marketing site to NestJS API with Turnstile token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Spoofing | Contact endpoint | mitigate | Turnstile server-side verification — client token never trusted without server verification |
| T-07-06 | Tampering | ZodValidationPipe | mitigate | contactSchema validates all fields (name max 100, email format, message max 5000, turnstileToken min 1) |
| T-07-07 | Denial of Service | POST /api/contact | mitigate | Rate limiting via existing @fastify/rate-limit config; 5 requests/10min per IP |
| T-07-08 | Information Disclosure | Contact form data | accept | Only name, email, company, message collected — no sensitive PII. HTTPS in transit. |
| T-07-09 | Tampering | Turnstile token expiry | mitigate | Token expires 300s after issuance. Form submits within window; Turnstile auto-generates new token on callback. |
</threat_model>
