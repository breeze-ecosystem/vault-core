<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Turnstile | Cloudflare Turnstile widget communicates with Cloudflare for bot detection |
| Browser → API | Form data POSTs to NestJS API with Turnstile token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-20 | Spoofing | TurnstileWidget | mitigate | Client-side widget renders; server-side verify is mandatory. Token never trusted without verification. |
| T-07-21 | Information Disclosure | ContactForm | mitigate | Form data transmitted over HTTPS; no sensitive/PII stored beyond what user submits voluntarily |
| T-07-22 | Denial of Service | POST to API | mitigate | Rate limiting handled server-side (5 req/10min per IP). Client shows error message on 429. |
| T-07-23 | Tampering | Form validation | mitigate | Client-side validation for UX; server-side Zod validation as security boundary |
| T-07-24 | Tampering | Turnstile token expiry | mitigate | Tokens expire after 300 seconds — form must be submitted within this window. If expired, Turnstile auto-generates new token on callback. |
| T-07-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages in this plan — uses existing React hooks and fetch API |
</threat_model>
