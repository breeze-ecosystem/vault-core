# Phase 7 — UI Design Contract

> Visual and interaction contract for the Public Presence phase. Generated by gsd-ui-researcher.
> Covers **WEB-01 through WEB-08**: a public-facing marketing website (`apps/marketing/`) on `oversighthub.com` as a separate Next.js 14 app — landing page, pricing page, MDX blog via velite, multi-language (6 locales) via next-intl, CSS-animated interactions with minimal `motion`, contact form with Cloudflare Turnstile, SEO with JSON-LD, and privacy-focused analytics.

### CSS Variable Architecture

The marketing site defines its own subset of CSS variables in `apps/marketing/app/globals.css` extending the `@repo/design` tokens. Unlike the dashboard (which uses `--shadcn-` prefix + Radix Themes vars), the marketing site uses a simpler set:

```css
/* apps/marketing/app/globals.css — token foundation */
@import url('https://fonts.googleapis.com/css2?family=IBM+Plex+Sans:wght@400;600&family=JetBrains+Mono:wght@400;500;600&display=swap');

@tailwind base;
@tailwind components;
@tailwind utilities;

@layer base {
  :root {
    /* Light-mode first (marketing default) */
    --marketing-bg: 0 0% 100%;               /* white */
    --marketing-bg-secondary: 210 20% 98%;   /* very light gray */
    --marketing-bg-card: 0 0% 100%;          /* white card surface */
    --marketing-bg-dark: 222 47% 11%;        /* dark navy for hero/CTA bands */
    --marketing-foreground: 222 47% 11%;     /* near-black text */
    --marketing-muted: 215 16% 47%;          /* secondary text */
    --marketing-muted-light: 215 20% 65%;    /* placeholder/caption text */
    --marketing-border: 214 32% 91%;         /* light borders, card borders */
    --marketing-border-strong: 215 20% 65%;  /* stronger borders */
    --marketing-primary: 190 90% 50%;        /* cyan — accent CTA color */
    --marketing-primary-hover: 190 90% 45%;  /* cyan hover state */
    --marketing-primary-foreground: 0 0% 100%; /* text on primary */
    --marketing-ring: 190 90% 50%;           /* focus rings */
    --marketing-success: 160 84% 39%;        /* form success */
    --marketing-destructive: 0 84% 60%;      /* form errors */
    --marketing-warning: 35 92% 50%;         /* warnings */
    --marketing-radius-sm: 0.375rem;
    --marketing-radius: 0.5rem;
    --marketing-radius-lg: 0.75rem;
    --marketing-radius-xl: 1rem;
    --marketing-container: 1280px;           /* max-width */
  }
}
```

**Why no dark mode on marketing?** Marketing site is a light-first brand extension (D-07). The dashboard handles dark/light mode internally. The marketing site may offer a dark theme toggle in a future phase, but for v2.0 it presents the brand in its premium light form.

## Typography

### Font Family

**IBM Plex Sans** (UI chrome: weights 400, 600) — same brand font as Dashboard. Loaded via Google Fonts in `globals.css`.

**JetBrains Mono** (weights 400, 500, 600) — reserved for blog code blocks and technical content (blog MDX prose, outside UI contract scope).

### UI Chrome Type Scale (4 sizes, 2 weights)

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| **Small** | 14px (text-sm) | 400 (regular) | 1.4 | Nav links, metadata, footnotes, section eyebrow labels, badges |
| **Body** | 16px (text-base) | 400 (regular) | 1.6 | Body paragraphs, card descriptions, feature lists, testimonial text, blog excerpts |
| **Body Emphasis** | 16px (text-base) | 600 (semibold) | 1.6 | CTA button text, active nav link, strong emphasis inline |
| **Heading** | 24px (text-2xl) | 600 (semibold) | 1.3 | Card titles, hero subtitle, section subheadings, pricing tier names, blog post titles |
| **Display** | 40px (custom) | 600 (semibold) | 1.1 | Hero headline, page H1 titles, major section headings |

**Exactly 4 sizes:** 14px, 16px, 24px, 40px.  
**Exactly 2 weights:** 400 (regular), 600 (semibold).

**Note — Blog MDX prose is outside this contract:** Blog post body content rendered via `@tailwindcss/typography` (prose plugin) uses a separate typographic scale with its own sizes and weights (including weight 700 for headings). That scale is managed entirely by the Tailwind typography plugin and is not part of the UI component contract.

**40px Display requires a Tailwind config extension.** Add to `apps/marketing/tailwind.config.ts`:
```ts
fontSize: {
  'display': ['40px', { lineHeight: '1.1', fontWeight: '600' }],
}
```

**Responsive behavior (Display):** 40px → 32px (tablet) → 24px (mobile).

**Key difference from Dashboard:** Marketing body text is 16px (vs 14px in dashboard) for readability in long-form narrative content. Headings use a dedicated 40px/24px pair (vs 20px in dashboard) for visual impact.

### Line Height Standardization
- Display: **1.1** — tight, impactful
- Heading: **1.3**
- Body: **1.6** — generous for readability
- Small: **1.4**
- CTA buttons: **1** (single line)

## Interaction Patterns

### Animation Philosophy

```
Animation Budget:
┌──────────────────────────────────────┬────────────┬──────────────┐
│ Layer                                │ Tool        │ Bundle Cost  │
├──────────────────────────────────────┼────────────┼──────────────┤
│ 90% — Simple interactions            │ CSS         │ 0 KB         │
│   (hovers, scroll reveals, micro-    │ transitions │              │
│    interactions, entrance animations) │ + keyframes │              │
├──────────────────────────────────────┼────────────┼──────────────┤
│ 10% — Complex transitions            │ `motion`    │ ~15KB gzipped│
│   (hero section carousel, testimonial│            │ (already in   │
│    carousel slide, page transitions) │            │  monorepo)    │
└──────────────────────────────────────┴────────────┴──────────────┘
```

### CSS Animations (90% of interactions)

| Element | Trigger | Effect | Duration | Easing |
|---------|---------|--------|----------|--------|
| Section entrance | Scroll into viewport | Fade up + translateY(24px → 0) | 600ms | `cubic-bezier(0.16, 1, 0.3, 1)` |
| Feature card hover | Mouse enter | Elevation increase (shadow deepen, translateY(-4px)) | 250ms | `ease-out` |
| CTA button hover | Mouse enter | Slight scale 1.03 + brightness 1.1 | 200ms | `ease-out` |
| CTA button active/press | Mouse down | Scale 0.97 | 100ms | `ease-in` |
| Nav link hover | Mouse enter | Underline slide-in (from left) | 200ms | `ease-out` |
| Pricing card hover | Mouse enter | Elevation increase, Professional card border glow intensify | 300ms | `ease-out` |
| Logo grayscale→color | Mouse enter on logo cloud item | Saturate from grayscale | 300ms | `ease-out` |
| Mobile hamburger menu | Click | Animated hamburger→X morph | 250ms | `ease-out` |
| Mobile menu panel | Open | Slide down + fade (max-height animation) | 300ms | `ease-out` |
| Language switcher hover | Mouse enter | Chevron rotate 180deg | 200ms | `ease-out` |
| Hero grid background | Always | Slow pan animation (subtle movement) | 20s loop | linear |
| Scroll-to-top button | Appear on scroll past fold | Fade in | 300ms | `ease-out` |
| Blog card hover | Mouse enter | Elevation increase + thumbnail subtle scale(1.02) | 300ms | `ease-out` |
| FAQ accordion toggle | Click | Max-height expand + chevron rotate | 300ms | `ease-out` |
| Form field focus | Focus | Border color transition to accent + subtle box-shadow | 200ms | `ease-out` |
| Form submission | Submit | Button loading spinner animation | loop | linear |
| Form success message | After submit | Slide down + fade in | 400ms | `ease-out` |
| Pricing feature checkmark | Render | Scale-in entrance | 300ms | spring |
| Skeleton loader (blog) | Page load | Shimmer animation (moving gradient) | 1.5s loop | linear |
| Trust bar scroll (mobile) | Auto | Horizontal scroll | 30s loop | linear |

### Motion Animations (10% — hero/carousel only)

| Element | Trigger | Effect | Duration | Easing |
|---------|---------|--------|----------|--------|
| Hero CTA entrance | Page load | Staggered fade-up: headline → subtitle → CTAs (stagger 150ms) | 600ms each | `ease-out` |
| Testimonial carousel slide | Nav click | Slide horizontal with fade | 400ms | `ease-out` |
| Showcase carousel item | Nav click | Fade + scale transition | 500ms | `ease-out` |
| Page scroll-to-section | Nav click | Smooth scroll to anchor (CSS `scroll-behavior: smooth` — no JS needed) | 600ms | `ease-out` |

### Reduced Motion

**All animations respect `prefers-reduced-motion`:**
- CSS: Wrap animations in `@media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } }`
- Motion: `import { useReducedMotion } from "motion/react"` and disable entrance sequences
- Hero grid background pauses on reduced motion
- Carousel becomes instant slides (no transition)
- Scroll reveals become instant appearance

### Responsive Navigation Pattern

**Desktop (>1024px):** Horizontal nav bar with logo, links, language switcher, CTA button. Transparent background on hero, solid on scroll (100px trigger).

**Tablet/Mobile (<1024px):** Hamburger menu icon replaces nav links. Slide-down panel on toggle. CTA collapses into mobile menu. Language switcher moves into mobile menu.

**Sticky header:** `position: sticky; top: 0; z-index: 50` — backdrop-blur effect on scroll for glass-like appearance.

## RTL Layout Contract (Arabic)

Full RTL support for Arabic locale per D-16:

| Element | RTL Behavior |
|---------|-------------|
| Layout direction | `<html dir="rtl">` applied by next-intl middleware for Arabic locale |
| CSS logical properties | Use `margin-inline-start` instead of `margin-left`, `padding-inline-end` instead of `padding-right`. Applies to all elements with directional spacing. |
| Flexbox direction | Flex containers auto-reverse with `dir="rtl"`. Verify grid/flex layouts render correctly. |
| Text alignment | Body text: `text-align: start`. Headings: `text-start`. |
| Icons with directional meaning | Flip icons that imply direction: arrow-left→arrow-right, chevron rotate, share icon. Use `rotate-180` for RTL via `[dir="rtl"] & .icon-arrow` CSS rule. |
| Language switcher | Flag/name ordering reversed in RTL mode. |
| FAQ accordion | Chevron rotates opposite direction in RTL. |
| Carousel | Slide direction reversed in RTL (right→left becomes left→right). `motion` animate values adjust for RTL. |
| Contact form | Labels align to the right. Submit button aligns to right. |
| Header logo | Logo on the right side. Nav links aligned to the left (in LTR logic, this is reversed automatically). |
| Footer grid | Column ordering reversed in RTL. |
| Blog post | Article text flows RTL. Code blocks remain LTR. |
| Focus ring | Focus order is DOM-order (unchanged). |
| Testing | Manually verify all pages with Arabic locale during implementation. Key pages: landing, pricing, blog, contact. |

**Tailwind RTL utilities available:** `ltr:`, `rtl:` prefixes. Use `ps-`/`pe-` (padding-inline-start/end) and `ms-`/`me-` (margin-inline-start/end) logical properties instead of directional `pl-`/`pr-` / `ml-`/`mr-`.

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| Keyboard navigation | All interactive elements reachable via Tab in logical order. Skip-to-content link as first focusable element. |
| Focus indicators | Cyan focus-visible ring on all interactive elements. Minimum 2px offset, 3px width. |
| Color contrast | Body text (#0f172a) on white (#ffffff): contrast ratio > 15:1. Cyan accent on white: > 3:1 minimum (for large text/UI components only). Cyan on dark navy: > 4.5:1. |
| Motion sensitivity | `prefers-reduced-motion` respected — all CSS animations disabled, scroll reveals become instant, carousel becomes static. |
| Screen reader | Semantic HTML landmarks (`<nav>`, `<main>`, `<footer>`, `<article>`, `<section>`). Proper heading hierarchy (h1→h2→h3). Alt text on all images. |
| Language switcher | `aria-label="Change language"` on switcher button. Each option has `lang` attribute matching locale. |
| RTL accessibility | Arabic pages use `dir="rtl"` — screen readers auto-switch reading direction. |
| Form accessibility | Labels associated with inputs via `htmlFor`. Required fields marked with `aria-required="true"`. Error messages linked via `aria-describedby`. |
| Links | Descriptive link text (never "click here"). External links have `aria-label` or visible indicator. |
| Images | All decorative images have `alt=""`. All meaningful images have descriptive `alt` text. Blog featured images use `alt` from frontmatter. |
| Skip link | "Skip to main content" as first tabbable element, visually hidden until focused. |
