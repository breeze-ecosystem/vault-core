    2. Create `forensic/forensic.service.ts`:
       - @Injectable() with Logger
       - Inject PrismaService, ConfigService
       - Method `async certifyEvidence(eventId: string, mediaType: "zip" | "clip", orgId: string): Promise<{ evidenceId: string; zipPath: string; tsaCertPath: string }>`:
         1. Generate unique evidenceId (crypto.randomUUID())
         2. Create temp directory `/tmp/evidence/{evidenceId}`
         3. Collect evidence data:
            - Fetch event metadata from DB (alert, incident, or access event)
            - Write metadata.json with: eventId, eventType, orgId, timestamp, cameraId, description
            - Find related media files (snapshots, clips) from filesystem paths
            - Copy media files to temp directory
         4. Create ZIP bundle of all evidence files via `execSync(cd {tempDir} && zip -r evidence.zip .)`
         5. Compute SHA-256 hash of ZIP: crypto.createHash("sha256").update(zipBuffer).digest("hex")
         6. Write hash to hash.txt
         7. Request RFC 3161 TSA timestamp:
            - TSA URL from config: `config.get<string>("TSA_URL", "http://timestamp.digicert.com")`
            - Step 1: `openssl ts -query -data {tempDir}/hash.txt -no_nonce -sha256 -out {tempDir}/tsq`
            - Step 2: `curl -s -H "Content-Type: application/timestamp-query" --data-binary @{tempDir}/tsq {tsaUrl} -o {tempDir}/tsr`
            - Handle TSA failure gracefully: if fails, log warning and create self-signed timestamp as fallback (openssl can generate non-authoritative timestamps)
         8. Bundle final output: `zip -r {outputPath} evidence.zip tsr hash.txt`
         9. Store ForensicEvidence record in Prisma:
            - id, eventId, organizationId, mediaType, zipPath, tsaCertPath, hash, sizeBytes
         10. Return { evidenceId, zipPath, tsaCertPath }
       - Method `async getEvidence(orgId: string, evidenceId: string): Promise<ForensicEvidenceDto>`:
         - Query Prisma by id + organizationId
         - Throw NotFoundException if not found
       - Method `async listEvidence(orgId: string, pagination): Promise<PaginatedResponse<ForensicEvidenceDto>>`:
         - Paginated list of evidence for org

    3. Create `forensic/forensic.controller.ts`:

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client→API | Storage management endpoints accessed via JWT-authenticated HTTP |
| API→NAS/external disk | CIFS mount + rsync via child_process shell commands |
| API→TSA service | RFC 3161 timestamp request via curl to external URL |
| API→Filesystem | Forensic evidence bundling in /tmp, storage in /mnt/evidence |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-17 | Tampering | Backup shell command injection | mitigate | Validate targetPath before mount: reject paths containing ;, |, $, backticks. Use execSync with timeout. |
| T-04-18 | Information Disclosure | Backup password storage | mitigate | NAS password stored in BackupConfig model. Field marked String? — consider pgp_sym_encrypt if sensitive. Password never returned in GET /config response. |
| T-04-19 | Denial of Service | Forensic evidence — disk space exhaustion | mitigate | Evidence stored in /mnt/evidence with configurable max size. BullMQ concurrency limits. Temp files cleaned up after certification. |
| T-04-20 | Spoofing | TSA timestamp — fake certificate | mitigate | Self-signed fallback clearly documented as non-authoritative. Primary DigiCert TSA URL configurable via TSA_URL env var. TSR file bundled with evidence for verification. |
| T-04-21 | Tampering | Forensic evidence — hash integrity | mitigate | SHA-256 hash computed before TSA timestamp. Hash stored in ForensicEvidence model. Evidence bundle includes hash.txt for independent verification. |
| T-04-22 | Information Disclosure | Multi-tenant backup data mixing | mitigate | Each org's backup uses separate mount point (/mnt/backup/{orgId}). rsync only copies org-scoped directories. BackupConfig is org-scoped via organizationId. |

</threat_model>
