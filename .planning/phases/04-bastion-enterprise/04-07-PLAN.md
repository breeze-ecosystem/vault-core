            - snapshotUrl: snapshot path

         6. Dispatch webhook: BASTION_EVENT_TYPES.FIRE_ALARM with full payload
         7. Send Socket.IO event to dashboard for real-time notification
         8. Create IntegrationEndpoint record if first event (or update lastEventAt)
         9. Return { alertId }
       - Method `async handleBmsEvent(payload: BmsEventPayload): Promise<{ alertId?: string }>`:
         1. Validate payload against bmsEventSchema
         2. Route by eventType (D-14 — event-based only, no bidirectional control):
            - hvac_temperature: if value exceeds thresholds (e.g., > 40°C or < 10°C), create WARNING alert
            - emergency_lighting: create INFO alert
            - fire_door_release: create CRITICAL alert with same video correlation as fire alarm
         3. For critical events: resolve nearest camera, capture snapshot
         4. Create alert if severity warrants (CRITICAL always, WARNING configurable)
         5. Dispatch webhook: BASTION_EVENT_TYPES.BMS_EVENT
         6. Send Socket.IO event
         7. Return { alertId } if alert created, or { alertId: undefined }
       - Private method `async resolveNearestCamera(siteId: string, zone: string)`:
         - Query camera where siteId matches and status="ONLINE"
         - Return first match (or null if none found)
       - Private method `async captureSnapshot(cameraId: string): Promise<string | null>`:
         - Use existing snapshot capture pipeline (reuse from access control correlation per D-13)
         - Return snapshot URL or null on failure
