## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Information Disclosure | Analytics export — data leakage across orgs | mitigate | All endpoints use orgId from JWT (req.user.orgId). TenantIsolationGuard enforces org scoping. BastionAnalyticsService queries always filter by orgId parameter. |
| T-04-07 | Tampering | Report generation — template injection | mitigate | Handlebars templates are static .hbs files, not user-controlled. Data passed to templates is sanitized Prisma query results. |
| T-04-08 | Denial of Service | Report generation — resource exhaustion | mitigate | BullMQ queue with concurrency limit. PDF generation is CPU-bound but runs in worker thread. Queue size limits prevent overload. |
| T-04-09 | Information Disclosure | Report data — unauthorized access | mitigate | All report endpoints require @Roles("ADMIN", "SUPER_ADMIN"). @RequiresPack("BASTION") gates entire module. Report store scoped by orgId. |
| T-04-10 | Repudiation | Audit missing on report generation | mitigate | All mutation endpoints use @Audited() decorator. Report generation logged with orgId, type, timestamp. |
