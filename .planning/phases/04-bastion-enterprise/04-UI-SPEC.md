## Typography

Canonical 4-size scale from `@repo/design/src/typography.ts`. All values apply in both dashboard (Tailwind) and mobile (StyleSheet) contexts.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 600 (semibold) | 1.2 | Page titles, KPI hero values |
| Heading | 20px | 600 (semibold) | 1.2 | Card titles, section headers, modal titles |
| Body | 14px | 400 (regular) | 1.5 | Paragraph text, table cells, form labels |
| Label | 12px | 600 (semibold) | 1.2 | Uppercase section titles, metric subtitles, stat labels, chart axes |

**Monospace convention:** Data-heavy readouts (percentages, counts, timestamps, retention values) use JetBrains Mono at the same size with `tabular-nums` + weight 600. Applied via `.stat-value` CSS class (existing: `font-mono tabular-nums text-2xl font-semibold tracking-tight`).

**Phase 4 additions:**
- Report metadata (dates, page numbers): `label` size + JetBrains Mono
- Export filenames: `body` size + JetBrains Mono
- HAPDP wizard progress indicators: `label` size + uppercase tracking-wider

## Interaction Patterns

### 1. HAPDP Wizard — Linear Multi-step Form

The HAPDP compliance wizard is the most complex new interaction. Requirements from BAS-30 to BAS-35.

**Flow:**
```
Step 1: Informations client       [auto-fill from org profile]
Step 2: Traitements déclarés      [select from system activity]
Step 3: Signalétique consentement [generate camera signage PDF]
Step 4: Pseudonymisation           [toggle on/off + preview]
Step 5: Portail accès sujet       [configure verification method]
Step 6: Récapitulatif + Export    [review all + download declaration PDF]
```

**Pattern:**
- Wizard container with fixed sidebar showing step indicators (number, name, status: pending/active/complete/error)
- "Suivant" / "Précédent" navigation buttons
- Each step validates before advancing
- Final step: "Tout est conforme — Générer la déclaration" primary CTA
- On completion: success animation (checkmark + details), PDF download trigger
- Persist progress in localStorage (auto-save after each step)

**States per step:**
| State | Visual |
|-------|--------|
| Not started | Circle ○ with step number |
| Active | Filled circle ● with cyan accent + pulsing glow |
| Complete | Checkmark ✓ with success green |
| Error | Warning ⚠ with red + inline error message |
| Skipped | Dashed circle - (for optional steps like consent signage) |

### 2. Subject Access Portal — Public 3-Step Flow

Requirements: BAS-34 (self-service: view/rectify/delete personal data).

**Flow:**
```
Step 1: Vérification identité   [email + code OTP]
Step 2: Consultation données    [display personal data overview]
Step 3: Action                  [download / rectifier / supprimer]
```

**Pattern:**
- Public route (no auth required), minimal layout (no sidebar)
- Identity verification via email OTP (existing notification service)
- Data display in read-only cards with "Signaler une erreur" and "Demander la suppression" CTAs
- Rectification → textarea form → admin approval workflow
- Deletion → confirmation dialog "Taper SUPPRIMER pour confirmer" → admin approval
- Success → confirmation email + reference number

### 3. Analytics Dashboard — Filtered Tab View

Requirements: BAS-20 to BAS-24.

**Layout:**
```
┌─────────────────────────────────────────────────────┐
│ PageHeader: "Analytique" + description               │
├─────────┬─────────┬──────────┬──────────┬───────────┤
│ KPI 1   │ KPI 2   │ KPI 3    │ KPI 4    │ KPI 5    │
│ Incidents│ Alertes │ Caméras  │ Stockage │ Entrées  │
│ today    │ actives │ en ligne │ utilisé  │ aujourd'hui│
├─────────┴─────────┴──────────┴──────────┴───────────┤
│ Filter bar: [Site ▼] [Période ▼] [Type d'événement] │
├─────────────────────────────────────────────────────┤
│ Tab: Tendances 7j | Tendances 30j | Export           │
├─────────────────────────────────────────────────────┤
│ LineChart / BarChart / AreaChart (responsive)        │
│                                                      │
│ ┌─────────────────────────────────────────────────┐  │
│ │ [Export CSV] [Export PDF] [Planifier rapport]   │  │
│ └─────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

**Interactions:**
- Site filter dropdown → reloads all charts for selected site
- Date range picker preset: "Aujourd'hui", "7 jours", "30 jours", "Personnalisé"
- Tab switch preserves filter state
- "Exporter CSV" → downloads filtered dataset
- "Exporter PDF" → generates report with current charts (loading state during generation)
- "Planifier rapport" → opens `report-schedule-config` modal

### 4. Auto-backup Configuration — Save + Test Pattern

Requirements: BAS-29.

- **Save flow:** Form fields → "Enregistrer" → validation → toast "Configuration sauvegardée"
- **Test flow:** "Tester la connexion" → spinner → success "NAS accessible" / error "[path] inaccessible, vérifiez les identifiants"
- **Manual trigger:** "Lancer une sauvegarde" → progress bar → completion toast + timestamp

### 5. Forensic Evidence Export — Certification Pipeline

Requirements: BAS-27.

- User selects event → "Exporter comme preuve judiciaire"
- Modal: evidence format (ZIP/video clip), evidence metadata form
- "Générer la preuve" → BullMQ job → progress animation (pulsing cert icon)
- Complete: "Preuve générée" + download button + TSA certificate info
- Error: "Échec de la certification" + retry option

### 6. Webhook Event Subscription — Category Toggle Groups

Requirements: BAS-42.

- Searchable list of event types grouped by category (Alerts, Access, AI, Compliance)
- Each event type: name + description + toggle switch
- "Tout sélectionner" / "Tout désélectionner" per category
- Save triggers webhook test delivery
