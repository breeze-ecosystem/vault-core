## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker volume → LicenseKeyManager | RSA private key file loaded from host-mounted volume |
| Codebase → application | RSA public key bundled as TypeScript constant in repo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | license-public-key.ts | accept | Public key is designed to be public — no mitigation needed. The private key NEVER enters the codebase. |
| T-05-02 | Tampering | docker-compose.prod.yml | mitigate | Volume mount uses `:ro` (read-only) to prevent container from modifying the key file. Documented in code comment. |
| T-05-03 | Tampering | .env.example / validation.ts | mitigate | LICENSE_PRIVATE_KEY_PATH is optional — if missing, LicenseKeyManager gracefully disables signing and logs a warning. This prevents crash-on-startup if key is not configured. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new npm packages in this plan — only `prisma` schema changes and TypeScript/JSON files. No package install gate needed. |
</threat_model>
