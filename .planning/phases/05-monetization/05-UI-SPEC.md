### Accent Reservation

**Accent (primary cyan) is reserved for:**
1. License status badge — `variant="default"` (cyan) for license is active + valid
2. Primary CTA buttons ("Créer une licence", "Activer la licence")
3. Progress bar fill for usage indicators (when under 80%)
4. Active/selected state in license table rows and nav items
5. Focus ring on license activation JWT input field
6. API key "Create" button and "Active" badge

**Do NOT use accent for:**
- Warning banners (use warning/amber)
- Error messages (use destructive/red)
- Expired license badges (use destructive/red)
- Disabled states (use muted gray)

## Page Structure

### Page 1: Admin License Management — `/licences`

**Route:** `app/(dashboard)/licences/page.tsx` (new route group or under existing admin section)

**Access:** `ADMIN` role (add to `nav-config.ts` in "Gouvernance" or new "Commercial" group)

**Layout:**
- `PageHeader` with title "Licences" and description "Gérez les licences des organisations"
- Action button: "Créer une licence" → opens `Dialog` with creation form
- Main content: `Table` listing all orgs' licenses

**Table columns:**
| Column | Type | Details |
|--------|------|---------|
| Organisation | Text | Organization name |
| Statut | Badge | `LicenseStatusBadge` — Active / Trial / Expirée / Grâce |
| Expire le | Date | With countdown via `LicenseExpiryCountdown` |
| Caméras | Progress | `LicenseUsageBars` showing current / max |
| Portes | Progress | `LicenseUsageBars` showing current / max |
| Actions | DropdownMenu | Remplacer / Révoquer / Voir détail |

**Create License Form (Dialog):**
1. Organisation select (searchable dropdown or native `<select>`) — list of all orgs
2. Nombre max de caméras (number input, default 4)
3. Nombre max de portes (number input, default 8)
4. Date d'expiration (date input with calendar picker)
5. Période de grâce en jours (number input, default 7)
6. Submit: "Générer la licence" → generates JWT, shows confirmation with copy-license-JWT button

**States:**
- **Empty:** `LicenseEmptyState` — "Aucune licence créée" / "Créez votre première licence pour un client"
- **Loading:** `Skeleton` table rows (5 rows)
- **Error:** Toast notification with error message

### Page 2: License Activation (Client) — `/licences/activation`

**Route:** `app/(dashboard)/licences/activation/page.tsx`

**Access:** All org users (visible via sidebar or reached from license status banner)

**Layout:**
- `PageHeader` with title "Activer votre licence" and description
- Card with `LicenseActivationForm`: large monospace textarea for JWT paste, "Activer" button
- On success: show decoded license claims (org name, device limits, expiry) in a success card
- On error: inline error message "Clé de licence invalide. Vérifiez la clé et réessayez."

**States:**
- **Before activation:** Form visible, paste area empty
- **Success:** Activation confirmation with license details
- **Invalid key:** Error state with inline message
- **Already activated:** If org already has a valid license, show message "Cette organisation a déjà une licence active" with link to status

### Page 3: License Status (Client Dashboard)

No dedicated page — license status displays in two places:

**A) Settings page** (`/parametres`): Add a "Licence" card at bottom of the settings page showing:
- `LicenseStatusBadge` (Active / Trial / Expirée)
- `LicenseExpiryCountdown`
- `LicenseUsageBars` for cameras and doors
- "Activer une licence" link (if no active license)
- "Gérer la licence" link (if active license, links to activation or detail)

**B) Dashboard header / Top bar**: Grace period warning banner:
- If in grace period: amber dismissible banner "Votre licence a expiré. Plus que X jours avant le mode lecture seule."
- If trial active: subtle cyan badge "Essai — X jours restants"
- If expired after grace: red banner "Licence expirée — Fonctionnalités limitées"

### Page 4: API Key Management

**Option A (recommended):** Dedicated section in licence route `/licences` — tab or collapsible section
**Option B:** Settings page `/parametres` — "Clés API" card

Choose A to keep license management co-located.

**Layout:** Card with header "Clés API pour génération de licences"
- `ApiKeyList` table: Name, Clé (••••••{last4}), Statut (badge), Créée le, Actions (Révoquer)
- "Créer une clé API" button → `ApiKeyCreateDialog`

- Nom de la clé (text input)

### Key User Flows

**Flow 1: Admin creates a license**
1. Admin navigates to `/licences`
2. Sees list (empty state if first time)
3. Clicks "Créer une licence"
4. Dialog opens with form
5. Fills in org, limits, expiry, grace period
6. Clicks "Générer la licence"
7. License JWT generated and displayed in result modal with copy button
8. Admin copies JWT, sends to customer (outside the app)
9. Table updates with new license row
10. Toast: "Licence créée avec succès"

**Flow 2: Client activates a license**
1. Customer navigates to `/licences/activation`
2. Pastes the JWT provided by admin
3. Clicks "Activer la licence"
4. System validates JWT signature, decodes claims, verifies org binding
5. Shows success with license details: limits, expiry date
6. Redirects to dashboard with new license status visible
7. Toast: "Licence activée avec succès"
8. If activation fails (invalid JWT, expired, wrong org): inline error message

**Flow 3: Over-limit detection (prevention + API)**
1. User reaches camera/door creation form
2. UI checks current usage from license status endpoint
3. If at limit: form shows upgrade prompt instead of submit button
4. "Limite atteinte — Vous avez atteint votre limite de X caméras. Contactez votre administrateur."
5. API also blocks creation for defense-in-depth (double barrier per D-15)

**Flow 4: Grace period → read-only transition**
1. License expires → 7-day grace period begins
2. Amber banner appears in dashboard header: "Votre licence a expiré. Période de grâce jusqu'au {date}."
3. All functionality continues working
4. After 7 days → system enters blocked mode
5. Red banner: "Licence expirée — Fonctionnalités en lecture seule"
6. API blocks all mutations (no new cameras, doors, etc.)

**Flow 5: Admin replaces a license**
1. Admin on `/licences` finds existing license row
2. Clicks "Remplacer" from row action dropdown
3. Opens create form pre-populated with current org and values
4. Admin adjusts limits and expiry, submits
5. New JWT generated, old one naturally expires on its original expiry
6. Customer can activate the new JWT on their dashboard

**Flow 6: API key management**
1. Admin navigates to API keys section on `/licences`
2. Clicks "Créer une clé API"
3. Enters key name, submits
4. Key shown once in copyable format with warning
5. Copies key, stores securely
6. Uses key for `POST /api/licenses/generate` endpoint
