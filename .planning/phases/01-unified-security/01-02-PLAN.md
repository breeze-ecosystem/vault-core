<objective>
Deliver the door monitoring and emergency response vertical slice. By the end of this plan, a security operator can view a real-time door status dashboard showing all door states per site, receive alerts for abnormal door states (held-open, forced, unsecured, desynchronized), and trigger emergency unlock or lockdown per zone. The door state machine with sequence-number-validated transitions and 500ms settling timeout prevents false alarms.

Purpose: This is the real-time physical security awareness layer. Door state is the most latency-sensitive stream in the system — operators must see door status changes within 1 second of the physical event. The state machine prevents the #1 failure mode of physical security dashboards: false door alarms that erode operator trust.

Output: Working door status dashboard with live Socket.IO updates, alert generation for all 4 abnormal states, and emergency override controls per zone.
</objective>
