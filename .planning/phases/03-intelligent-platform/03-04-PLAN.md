<objective>
Deliver the Predictive Equipment Health vertical slice — extending the Phase 2 EquipmentService with trend analysis for predicting equipment degradation, plus camera-to-door visualization. By the end of this plan, the system predicts battery drain rates, FPS degradation, and connection stability issues using simple linear regression on TimescaleDB health time-series data, and provides a camera-to-door association map that detects mismatches and gaps.

Purpose: Moving from reactive equipment monitoring (Phase 2) to predictive maintenance (Phase 3). Instead of alerting when equipment fails, the system alerts before failure occurs. Camera-to-door visualization helps operators verify their security coverage is complete.

Output: Working predictive health engine with hourly trend analysis, predictions hypertable for trend data, REST API for prediction queries, and two dashboard pages (predictions view and camera-to-door map).
</objective>

    ```sql
    CREATE TABLE predictions (
        time TIMESTAMPTZ NOT NULL,
        site_id UUID NOT NULL,
        device_type VARCHAR(32) NOT NULL,  -- 'camera', 'reader', 'controller'
        device_id UUID NOT NULL,
        metric VARCHAR(64) NOT NULL,        -- 'battery_level', 'fps_ratio', 'latency_ms', 'failed_reads'
        current_value DOUBLE PRECISION NOT NULL,
        failure_threshold DOUBLE PRECISION NOT NULL,
        slope DOUBLE PRECISION NOT NULL,     -- Rate of change per hour
        hours_to_failure INTEGER,            -- Estimated hours until threshold crossed (NULL if not degrading)
        confidence VARCHAR(16) NOT NULL,      -- 'high', 'medium', 'low'
        data_points INTEGER NOT NULL,         -- Number of data points used in trend
        triggered_alert BOOLEAN DEFAULT FALSE
    );

    SELECT create_hypertable('predictions', 'time',
        chunk_time_interval => INTERVAL '7 days'
    );

    ALTER TABLE predictions SET (
        timescaledb.compress,
        timescaledb.compress_segmentby = 'site_id, device_type',
        timescaledb.compress_orderby = 'time DESC'
    );

    SELECT add_compression_policy('predictions', INTERVAL '30 days');

    SELECT add_retention_policy('predictions', INTERVAL '365 days');

    CREATE INDEX idx_predictions_device_time ON predictions (device_type, device_id, time DESC);
    CREATE INDEX idx_predictions_site_time ON predictions (site_id, time DESC);
    CREATE INDEX idx_predictions_pending ON predictions (triggered_alert, time DESC) WHERE hours_to_failure IS NOT NULL;
    ```

    Class structure:
    ```typescript
    @Injectable()
    export class EquipmentPredictor {
      private readonly logger = new Logger(EquipmentPredictor.name);

      // Minimum data points required for a prediction
      private readonly MIN_DATA_POINTS = 10;
      // Minimum trend duration in days before alerting
      private readonly MIN_TREND_DAYS = 3;
      // Failure thresholds
      private readonly THRESHOLDS = {
        batteryLevel: 5,     // 5% = failure
        fpsRatio: 0.3,       // 30% of expected FPS = failure
        latencyMs: 5000,     // 5000ms = failure
        failedReads: 50,     // 50 failed reads per hour = failure
      };

      constructor(
        private prisma: PrismaService,
        private eventEmitter: EventEmitter2,
      ) {}
    ```

    Methods:

    1. `predictTrend(readings: { time: Date; value: number }[], metric: string, failureThreshold: number): TrendResult`:
       - Private method implementing simple linear regression (RESEARCH.md lines 631-656)
       - Compute slope (rate of change per reading), normalize to per-hour
       - Calculate hours-to-failure if slope indicates degradation toward threshold
       - Confidence based on data point count: >50 'high', >20 'medium', else 'low'
       - Return `{ slope, currentValue, failureThreshold, hoursToFailure, confidence, dataPoints }`
       - If not enough data (< MIN_DATA_POINTS), return null
       - If not degrading (slope >= 0), return null (no failure predicted)

    2. `predictControllerBatteryFailure(controllerId: string, siteId: string): Promise<TrendResult | null>`:
       - Query last 7 days of controller_health for battery_level readings
       - Call predictTrend() with battery_level threshold of 5%
       - Return result or null

    3. `predictCameraFpsFailure(cameraId: string, siteId: string): Promise<TrendResult | null>`:
       - Query last 7 days of camera_health for fps_actual readings
       - Compute fps_ratio = fps_actual / fps_expected per reading
       - Call predictTrend() with fps_ratio threshold of 0.3 (30% of expected)
       - Return result or null

    4. `predictReaderFailure(readerId: string, siteId: string): Promise<TrendResult | null>`:
       - Query last 7 days of reader_health for failed_reads readings
       - Call predictTrend() with failed_reads threshold of 50 (trending UP is bad)
       - Note: for failed_reads, higher values = worse. Adjust hours-to-failure accordingly
       - Return result or null

    5. `computeAllPredictions(): Promise<void>`:
       - Called by cron (via EquipmentService)
       - Fetch all active cameras, readers, and controllers
       - For each device, compute the relevant prediction
       - If prediction exists and hoursToFailure <= 72 (3 days): emit `prediction.triggered` event
       - Write each prediction to the predictions hypertable via $queryRawUnsafe
       - Log summary: "Predictions computed for N devices, X failures predicted"
