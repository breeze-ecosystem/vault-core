    1. Create the risk_level enum and hypertable:
    ```sql
    CREATE TYPE risk_level AS ENUM ('low', 'moderate', 'elevated', 'critical');

    CREATE TABLE risk_scores (
        time TIMESTAMPTZ NOT NULL,
        zone_id UUID NOT NULL,
        site_id UUID NOT NULL,
        score SMALLINT NOT NULL CHECK (score >= 0 AND score <= 100),
        risk_level risk_level NOT NULL,
        factors JSONB NOT NULL,
        smoothed_score SMALLINT NOT NULL CHECK (smoothed_score >= 0 AND smoothed_score <= 100)
    );

    SELECT create_hypertable('risk_scores', 'time',
        chunk_time_interval => INTERVAL '7 days'
    );

    ALTER TABLE risk_scores SET (
        timescaledb.compress,
        timescaledb.compress_segmentby = 'site_id, zone_id',
        timescaledb.compress_orderby = 'time DESC'
    );

    SELECT add_compression_policy('risk_scores', INTERVAL '14 days');

    SELECT add_retention_policy('risk_scores', INTERVAL '90 days');

    CREATE INDEX idx_risk_scores_zone_time ON risk_scores (zone_id, time DESC);
    CREATE INDEX idx_risk_scores_site_time ON risk_scores (site_id, time DESC);
    CREATE INDEX idx_risk_scores_level ON risk_scores (risk_level, time DESC) WHERE risk_level IN ('elevated', 'critical');
    ```

    **Part B — Risk Scoring Engine (risk.service.ts):**

    Create `apps/api/src/modules/risk/risk.service.ts` following RESEARCH.md Pattern 2 (lines 349-486) EXACTLY.

    Class structure:
    ```typescript
    @Injectable()
    export class RiskService implements OnModuleInit {
      private readonly logger = new Logger(RiskService.name);

      // Weight configuration (all configurable via properties for future env-var override)
      private readonly WEIGHTS = {
        deniedAttempts: 25,
        openDoorAnomalies: 30,
        anomalyEvents: 25,
        activeIncidents: 15,
        failedReaders: 5,
      };

      // Smoothing factor for Pitfall 2 mitigation (prevents oscillation)
      private readonly SMOOTHING_ALPHA = 0.3;

      constructor(
        private prisma: PrismaService,
        private eventEmitter: EventEmitter2,
        @Inject("REDIS_RISK") private redis: Redis,
      ) {}
    ```

    Methods:

    1. `onModuleInit()`: Load previous scores from risk_scores hypertable for each zone (last entry per zone) to have baseline for smoothing on first cron run. Log scoring engine started.

    2. `@Cron("*/5 * * * *") computeAllZoneScores()`:
       - Fetch all active zones from Prisma with site relation
       - For each zone, collect risk factors via `collectRiskFactors()`
       - Calculate raw score via `calculateScore(factors)`
       - Load previous smoothed score from Redis or risk_scores hypertable
       - Apply exponential smoothing: `smoothed = α × raw_score + (1 - α) × previous_smoothed` (Pitfall 2 mitigation)
       - Insert into risk_scores hypertable via $queryRawUnsafe
       - Cache in Redis: `risk:zone:{zoneId}` with 10-min TTL
       - If score >= 70 (critical): emit `risk.score-critical` event via EventEmitter2
       - Log summary: "Risk scores computed for N zones, X critical, Y elevated"
       - Wrap entire method in try/catch with error logging to prevent cron from silently dying

    3. `calculateScore(factors: RiskFactors): number`:
       - Raw = min(denied/10, 1) * 25 + min(anomalies/5, 1) * 30 + min(anomalyEvents/10, 1) * 25 + min(incidents/3, 1) * 15 + min(failedReaders/2, 1) * 5
       - Recency bonus: hoursSinceLastEvent < 1 → +10, < 6 → +5, < 24 → +2, else 0
       - Return min(100, round(raw + recency_bonus))

    4. `classifyLevel(score: number): string`: >=70 → "critical", >=40 → "elevated", >=20 → "moderate", else "low"

    5. `collectRiskFactors(zoneId: string, siteId: string): Promise<RiskFactors>`:
       - Parallel queries:
         - Count denied access events in last 24h for this zone
         - Count door anomalies (forced, held-open) from door_state_log for doors in this zone in last 24h
         - Count AI anomaly events (from alerts where metadata->>'type' IN ('intrusion','loitering','tailgating') and created in last 24h)
         - Count active (not resolved/closed) incidents for this zone
         - Count readers with >10 failed reads in last 24h for this site (from reader_health hypertable)
         - Get hours since last event (max of latest access_event, door_state_log, alert timestamp)
       - All queries via $queryRawUnsafe
       - Return RiskFactors

    6. `getCurrentScores(siteId?: string): Promise<RiskScoreDto[]>`:
       - Read from Redis cache first (keys matching `risk:zone:*`)
       - Fallback: query risk_scores hypertable for latest score per zone (DISTINCT ON)
       - Join with zones table for zone name, sites table for site name
       - Return RiskScoreDto[]

    7. `getZoneScoreHistory(zoneId: string, from?: string, to?: string): Promise<RiskTrendPoint[]>`:
       - Query risk_scores hypertable for the zone over the time range
       - Return ordered trend points

    8. `getSiteRiskSummaries(): Promise<SiteRiskSummary[]>`:
       - Aggregate current scores per site: average, max, count of critical/elevated zones
       - Return summaries
