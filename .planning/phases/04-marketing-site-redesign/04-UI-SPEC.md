## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, glass panel inner margins |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, card content padding |
| lg | 24px | Section internal padding, card groups |
| xl | 32px | Layout gaps between major blocks |
| 2xl | 48px | Major section breaks, hero to next section |
| 3xl | 64px | Page-level spacing, section top/bottom padding |

Exceptions: `py-24` (96px) and `py-32` (128px) allowed for section outer vertical padding to achieve the premium "breathe heavily" feel per high-end-visual-design skill. This is a macro-whitespace override at the section container level, not a scale change.

**Source:** Default 8-point scale recommended by template. Aligns with `packages/design/src/spacing.ts` (differences: md standardized to 16px from 12px). Confirmed by user.

## Interaction & Animation Contract

| Pattern | Implementation | Scope |
|---------|---------------|-------|
| Scroll-triggered fade-up | `motion` `whileInView` with `translateY(24px)` → `translateY(0)`, `opacity(0)` → `opacity(1)`, ~700ms `cubic-bezier(0.16, 1, 0.3, 1)` | Hero entrance, section titles, feature cards, stats counters, CTA sections (D-15, D-16) |
| Hover scale on cards | `hover:scale-[1.02]` or `group-hover:scale-105` with `transition-transform duration-300` | Feature cards, case study cards, pricing cards |
| Button press | `active:scale-[0.97]` with `transition-transform duration-150` | All interactive buttons (existing pattern in `button.tsx`) |
| Hero entrance stagger | `motion` staggered children with increasing `delay` (0s, 0.15s, 0.3s, 0.5s) | Hero headline, subtitle, CTAs, trust text |
| Stats counter | Animate from 0 to final value on scroll into view using `motion` `useInView` + `animate` | Stats section numbers |
| Sticky nav transition | Non-scrolled: transparent. Scrolled: `bg-black/60 backdrop-blur-2xl border-b border-white/5` with 300ms transition | Header glass morph on scroll |
| Mobile menu reveal | Full-screen overlay with `backdrop-blur-3xl bg-black/80`, nav links staggered fade-up (delay 100/150/200ms) | Mobile hamburger menu |
| Glassmorphism panels | `bg-white/[0.03] backdrop-blur-xl border border-white/[0.06] rounded-2xl` | Feature cards, demo tour tooltips, testimonial cards, pricing cards |
| Animated grid background | CSS `background-image` grid pattern with cyan/teal at ~0.04 opacity, animated via `motion` | Hero section |
| Noise texture overlay | Fixed `pointer-events-none` pseudo-element with CSS noise SVG at opacity 0.03–0.04 | Page-level texture depth |
| Section divider | Subtle gradient line between sections | Visual separation |

**Performance constraints per high-end-visual-design skill:**
- Animate only `transform` and `opacity` — never `top`, `left`, `width`, `height`
- `backdrop-blur` only on sticky/fixed elements (nav, overlays), never on scrolling containers
- Noise texture as fixed `pointer-events-none` pseudo-element, not attached to scrolling content
- Use `IntersectionObserver` / `motion` `whileInView` for scroll reveals — never `window.addEventListener('scroll')`

**Reduced motion:** Respect `prefers-reduced-motion: reduce` — set all animation-duration to 0.01ms (existing pattern in `globals.css`).

**Source:** CONTEXT.md D-06 (mixed hero, 1-2 WOW moments), D-15 (subtle micro-animations), D-16 (major section pieces only). High-end-visual-design skill for implementation patterns. Existing `tailwind.config.ts` animations preserved and extended.
