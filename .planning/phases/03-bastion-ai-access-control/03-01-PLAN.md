## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client→AI Preprocessor | Untrusted frames submitted for detection via HTTP |
| AI Preprocessor→NestJS | Detection results dispatched via internal HTTP (fire-and-forget) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | YOLO model weights | mitigate | Model weights loaded from protected filesystem. File integrity checked on container start via SHA-256 hash against pinned manifest. |
| T-03-02 | Spoofing | Anti-spoofing bypass (photo/video) | mitigate | Passive liveness with multi-cue analysis. Per D-15: low liveness score = detection disregarded + operator alert. Configurable thresholds per org. |
| T-03-03 | Information Disclosure | Face embedding cache in AI Preprocessor memory | mitigate | In-memory cache never exposed externally. Cache refresh via authenticated internal API only. No raw embeddings in logs per AI-SPEC §4b.1. |
| T-03-04 | Denial of Service | AI Preprocessor CPU exhaustion from crafted frames | mitigate | Per-frame timeout at 500ms (guardrail #5 in AI-SPEC §6). Frame dropped rather than queued. 3-of-5 temporal smoothing prevents single-frame attacks. |
| T-03-05 | Elevation of Privilege | Blacklist match false positive → armed response | mitigate | Configurable risk threshold (D-11), uncertain range (60-84 logs + operator notify). Blacklist CRITICAL alert requires both match_score >= threshold AND liveness_score > 0.6. |
| T-03-06 | Tampering | Detection model adversarial patch | mitigate | Temporal smoothing (3-of-5 frames) mitigates single-frame adversarial patches per AI-SPEC domain research. |
| T-03-07 | Spoofing | Weapon detection false positive from tool/reflection | mitigate | Higher confidence threshold for weapons (0.6 per D-21). Zone-restricted detection (guardrail #7). Low-confidence (0.3-0.59) sent to review queue only. |
| T-03-SC | Tampering | Python package legitimacy | mitigate | No new packages required. All packages already in requirements.txt and Dockerfile per RESEARCH.md Package Legitimacy Audit. |
