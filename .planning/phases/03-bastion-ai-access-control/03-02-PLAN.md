## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08 | Information Disclosure | Face model stores photoBase64 | mitigate | Photo stored as base64 on local filesystem only (via @fastify/static). Not exposed via API response by default. Access gated by role (ADMIN+). |
| T-03-09 | Tampering | Qdrant faces collection | mitigate | Qdrant API not exposed externally. All operations through NestJS auth chain (JwtAuthGuard + RolesGuard). Face operations require ADMIN+ role plus RequiresModule("advanced_facial_recognition"). |
| T-03-10 | Elevation of Privilege | GLOBAL_ADMIN role misuse | mitigate | TenantIsolationGuard extended in Plan 03 to restrict child org access to declared sub-sites only. GLOBAL_ADMIN cannot access child org data without explicit site admin grant. |
| T-03-11 | Tampering | CredentialType enum injection | mitigate | Type-safe enum values in Prisma. Zod validation on API layer rejects invalid types. Existing BadRequestException patterns from access.service.ts reused. |
| T-03-12 | Spoofing | FINGERPRINT credential type — template hash replay | mitigate | Per D-23: fingerprint verification happens on-reader only. Template hash stored is a reference identifier, not a verifiable biometric. No raw biometric data transferred to vault-os. |
| T-03-SC | Tampering | npm package legitimacy | mitigate | No new npm packages required. @prisma/client, @repo/shared, @nestjs/common already in use. |
