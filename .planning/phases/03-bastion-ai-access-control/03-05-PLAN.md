## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser→API | All credential, face, and access operations via JWT |
| FaceUploadDropzone→API | Face photo upload for enrollment (base64) |
| CredentialForm→API | Credential creation with sensitive data (badge numbers, PINs) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-25 | Information Disclosure | Face photo base64 in page HTML | mitigate | Photo loaded as image src, not inline base64 in HTML (unless explicitly for upload preview). API responses exclude photoBase64 from list endpoints, only include in detail endpoint (role-gated). |
| T-03-26 | Tampering | Credential type field manipulation | mitigate | Zod validation on API rejects invalid types. UI type selector renders only allowed options per user role. |
| T-03-27 | Information Disclosure | Access event timeline exposes PII | mitigate | Events shown only to authorized roles (SUPERVISOR+). Snapshot access gated by same role. Sensitive data (badge numbers, full PINs) masked in UI — show last 4 digits only. |
| T-03-28 | Tampering | Delete credential without confirmation | mitigate | Delete requires typing "SUPPRIMER" text input per UI-SPEC destructive action pattern. Confirmation dialog always shown. @Audited() on API side. |

</threat_model>
