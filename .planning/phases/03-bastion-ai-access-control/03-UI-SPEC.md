## Spacing Scale

From `@repo/design/src/spacing.ts` — canonical 9-step scale. All values are multiples of 4.

| Token | px | Usage |
|-------|----|-------|
| xs | 4 | Icon gaps, badge padding, status dot spacing |
| sm | 8 | Compact element spacing, button icon gaps, table cell padding |
| md | 12 | Form field spacing, card inner padding (compact), list gaps |
| base | 16 | Default element spacing, card padding, section heading margin |
| lg | 24 | Section padding, grid gaps, sub-section separation |
| xl | 32 | Layout section margins, modal/content padding, sidebar group gaps |
| xxl | 48 | Major section breaks, page padding top/bottom |
| xxxl | 64 | Page-level spacing between large content blocks |
| xxxxl | 96 | Hero spacing, full-page empty states |

**Exceptions for Phase 3:**
- Multi-site KPI grid cards: use `gap-4` (16px) between cards, `p-5` inside cards (matching existing `StatsCard` pattern)
- Access schedule grid cells: use `sm` (8px) for compact day/hour cell rendering
- Face grid cards: use `gap-4` (16px) grid gap, `p-3` (12px) inner padding (matching existing `visages` page pattern)
- Touch targets in mobile: minimum 44×44px for icon-only buttons in access lists and face enrollment
- Mobile table row height: minimum 48px for tap targets in credential lists

### Accent Reserved For

The accent color (cyan, #06b6d4) is reserved EXCLUSIVELY for:
1. **Primary buttons** — `variant="default"` (main CTAs, form submit)
2. **Focus rings** — `ring-2 ring-primary` (keyboard navigation, active form fields)
3. **Active/toggled state** — selected navigation item, active tab underline, switch ON state
4. **Online/active status indicators** — camera online pulse, operator active dot, connected badge
5. **High-severity alert indicators** — CRITICAL alert badge (for AI detection matches)
6. **Key metric highlights** — primary KPI value in multi-site dashboard (total cameras, live alerts)
7. **Face match confirmation** — matched face border highlight, 85+ risk score badge
8. **Data visualization** — primary data series line in cross-site comparison charts
9. **Loading/progress** — progress bar fill, indeterminate animation
10. **Global search** — search results count badge, active search filter

Do NOT use accent for: secondary buttons, static labels, decorative elements, informational badges, disabled states, card borders (use `border-border`), default text.

### Face Enrollment Flow (BASTION)

```
1. Page: /visages → List all enrolled faces (grid cards)
2. CTA: "Ajouter un visage" → Dialog opens
3. Step 1: FaceUploadDropzone → select/capture photo
4. Step 2: Name input + optional metadata
5. Step 3: Blacklist toggle (if enabled → risk threshold slider)
6. Step 4: Auto-unlock config per door (Bas-01, D-09):
   [x] Activer le déverrouillage auto
   Seuil de confiance: [═══════●══════] 85
   Portes concernées: [Porte principale ▼] [+ Ajouter]
7. Submit → AI Preprocessor enrolls embedding in Qdrant
8. Toast: "Visage enrôlé avec succès"
9. Face card appears in grid with recognition badge + risk score display
```

### Access Credential Provisioning Flow

```
1. Page: /acces → List all credentials (DataTable)
2. CTA: "Nouveau justificatif" → Form page
3. Type selector [Tabs]: Badge RFID | Empreinte | QR Code | PIN
4. Form fields per type:
   - Badge: badge number (manual entry), reader assignment
   - Fingerprint: scan at reader → template hash returned → auto-associate
   - QR: generate or enter numeric code
   - PIN: numeric PIN with confirmation
5. Owner: search/create user from dropdown
6. Site assignment: which sites this credential works on
7. Schedule: link to existing schedule or create new
8. Group membership: select role group
9. Expiry: date range picker + optional max uses
10. Submit → creates credential + optional publish to MQTT
11. Toast: "Justificatif créé"
```
