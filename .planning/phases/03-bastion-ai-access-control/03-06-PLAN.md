<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Mobile→API | All mobile operations via JWT-authenticated HTTP |
| Mobile camera→local storage | Face photo captured and held in memory before upload |
| AI Preprocessor test→pipeline | Direct test of detection endpoints with synthetic frames |
| API test→services | Unit tests with mocked dependencies |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-29 | Information Disclosure | Mobile face photo in memory | mitigate | Photo held in component state only during enrollment. Cleared on submit or cancel. Not persisted to device storage. expo-secure-store not used for enrollment data. |
| T-03-30 | Spoofing | Camera face enrollment — photo of photo | mitigate | Anti-spoofing runs on AI Preprocessor side during enrollment. AI Preprocessor analyzes the uploaded photo for liveness; low-liveness enrollment photos are rejected. |
| T-03-31 | Information Disclosure | Access events on mobile display PII | mitigate | Events require authentication (JWT in SecureStore). Same role-based access as dashboard. Snapshot access gated by same API authorization. Biometric/PIN app lock recommended but not enforced at API level. |
| T-03-32 | Elevation of Privilege | Site switcher shows unauthorized sites | mitigate | Site list fetched from API (GET /api/sites), which enforces TenantIsolationGuard. Mobile only renders what API returns. No client-side site enumeration possible. |

</threat_model>
