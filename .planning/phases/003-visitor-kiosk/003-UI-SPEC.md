# Phase 3 вҖ” UI Design Contract

> Visual and interaction contract for Phase 3: Visitor Kiosk. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Key design principle:** The kiosk is a standalone touchscreen SPA served by nginx in a Docker container. It is NOT the Dashboard вҖ” no shadcn/ui, no dark theme, no sidebar navigation. Light, clean, welcoming lobby aesthetic with large touch targets (48px+). French primary with English toggle. Auto-idle resets to welcome screen.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Tailwind CSS (kiosk-specific config at `apps/kiosk/`) |
| Preset | Not applicable вҖ” no shadcn. kiosk uses custom Tailwind config with light theme tokens. |
| Component library | Custom (no Radix UI). Build: Button, Card, Input, Dialog, CameraOverlay. |
| Icon library | lucide-react (same as Dashboard, already in monorepo) |
| Font | System font stack: `system-ui, -apple-system, "Segoe UI", Roboto, Helvetica, Arial, sans-serif` вҖ” no external font requests, native tablet feel. Monospace: `"SF Mono", "Fira Code", "Fira Mono", "Roboto Mono", monospace` for technical references. |
| Animation | CSS transitions only (no `motion` вҖ” kiosk runs on low-power tablets, keep animations minimal). `transition-all duration-200` for screen transitions, `animate-pulse` for printing indicator. |

**Key constraint:** `apps/kiosk/` is a NEW app in the monorepo. It reuses `@repo/shared` types and ESLint config but has its own `tailwind.config.ts`, `globals.css`, and component tree. No dependency on Dashboard components.

## Spacing Scale

All values follow Tailwind defaults (4px base). Verified for touch-friendly layout.

| Token | Tailwind | Value | Usage |
|-------|----------|-------|-------|
| xs | `gap-1` / `p-1` | 4px | Inline icon-to-text gaps, QR code padding |
| sm | `gap-2` / `p-2` | 8px | Button icon spacing, badge element gaps |
| md | `gap-4` / `p-4` | 16px | Card content padding, form field spacing |
| lg | `gap-6` / `p-6` | 24px | Section padding, screen vertical gaps |
| xl | `gap-8` / `p-8` | 32px | Welcome screen centering, major layout gaps |
| 2xl | `gap-12` / `p-12` | 48px | Kiosk full-screen padding (lobby tablet) |
| 3xl | `gap-16` | 64px | Welcome screen vertical spacing between hero and CTAs |

**Touch target exceptions (WCAG 2.5.8 Target Size):**
- **Primary CTAs** (`Scan QR`, `Search by name`, `Confirmer`, `Imprimer`): 56px minimum height (Tailwind `h-14`) вҖ” larger than standard 48px for lobby usability
- **Secondary buttons** (`Annuler`, `English`/`FranГ§ais` toggle, `RГ©essayer`): 48px minimum height (Tailwind `h-12`)
- **QR scanner viewfinder**: full-width Г— 280px tall camera area
- **Name search input**: 56px height with 18px text (easier to tap/type on tablet)
- **Language toggle**: 44px Г— 44px minimum touch target (WCAG compliant)

## Typography

System font stack means no explicit font loading. Sizes are optimized for tablet-sized screens (10-13" displays viewed at arm's length).

| Role | Size | Tailwind | Weight | Line Height | Usage |
|------|------|----------|--------|-------------|-------|
| Display | 28px | `text-3xl` | 700 (bold) | 1.2 | Welcome screen heading ("Bienvenue"), screen titles, visitor name display, section headers |
| Body | 18px | `text-lg` | 400 (regular) | 1.5 | Visitor info fields, scan instructions, body text, button text, form labels |
| Label | 16px | `text-base` | 400 (regular) | 1.4 | Metadata, secondary info, timestamps, error codes |
| Caption | 14px | `text-sm` | 400 (regular) | 1.4 | Status messages, printer status, hints, QR data text, technical identifiers |

**Font weights used:** 400 (regular), 700 (bold) вҖ” only two weights used. 44px brand treatment moved to logo graphic (36px `text-4xl` bold or SVG logo).

**Line-height note:** 1.2 on display for tight heading spacing; 1.5 on body for readability on tablet screens.

## Color

Light/clean palette вҖ” professional, welcoming, high-contrast for lobby/public space. Uses hardcoded hex values via Tailwind config (no CSS custom properties, no dark mode).

Dominant surface is white (60%). Secondary is warm grey (30%). Accent is a professional blue (10%).

| Role | Hex | Tailwind | Usage |
|------|-----|----------|-------|
| Dominant (60%) | `#FFFFFF` | `bg-white` | Full-screen background, card surfaces, all main content areas |
| Secondary (30%) | `#F8F9FA` | `bg-gray-50` | Welcome screen background band, container cards, modal overlays |
| Surface accent | `#F0F4F8` | `bg-blue-50` | QR scanner container, confirmation section highlight, search results highlight |
| Primary text | `#1A1D23` | `text-gray-900` | Headings, body text, labels вҖ” high contrast |
| Secondary text | `#6B7280` | `text-gray-500` | Status messages, hints, timestamps, captions |
| Accent (10%) | `#2563EB` | `text-blue-600` / `bg-blue-600` | Primary CTA buttons, active state, focus rings, QR scan highlight |
| Accent hover | `#1D4ED8` | `text-blue-700` / `bg-blue-700` | CTA hover state |
| Success | `#059669` | `text-emerald-600` / `bg-emerald-600` | Check-in success, "Printed" badge, checkmark icon |
| Error | `#DC2626` | `text-red-600` / `bg-red-600` | Check-in failure, printer error, scan failure |
| Warning | `#D97706` | `text-amber-600` / `bg-amber-600` | Printer out-of-paper, low ink, scan retry prompt |
| Border | `#E5E7EB` | `border-gray-200` | Card borders, input borders, divider lines |
| Input border | `#D1D5DB` | `border-gray-300` | Input field borders, focus transitions to accent |
| Overlay | `rgba(0,0,0,0.4)` | `bg-black/40` | Modal backdrop, camera overlay background |

### Accent (10%) вҖ” Reserved For:
- **Primary CTA buttons** (`Confirmer`, `Imprimer`, `Scan QR`): solid `bg-blue-600` with white text
- **Active/selected search result**: `bg-blue-50` with `border-blue-500` left border
- **Focus rings**: `ring-blue-500` on inputs and buttons during keyboard/voice navigation
- **QR scanning success indicator**: blue border overlay on detected QR code
- **Language toggle active**: blue underline or `text-blue-600` on selected language
- **Visitor confirmation section**: subtle `bg-blue-50` background for photo + details area
- **Touch feedback**: `active:bg-blue-700` on primary buttons

### Success (check-in complete):
- Success screen: large green checkmark `text-emerald-500`, green header background band
- "Badge printed" confirmation: green badge `bg-emerald-100 text-emerald-700`
- Check-out confirmation: green flash overlay

### Error (failure states):
- Printer error: red icon `text-red-500`, red background `bg-red-50`, "RГ©essayer" button
- QR scan failure: red border flash on viewfinder, error message card
- API error (network/printer): red banner at top with problem description + retry path

### Destructive actions:
- None in this phase вҖ” no destructive actions exist in the kiosk flow.

## Screen State Machine

The kiosk SPA uses a React state machine (not Next.js App Router navigation). A single `kioskPhase` state drives which component renders.

```typescript
type KioskPhase = 
  | "welcome"
  | "scanning" 
  | "search" 
  | "confirm" 
  | "printing" 
  | "success" 
  | "checkout-success" 
  | "error";
```

Transitions:
- `welcome` вҶ’ `scanning`: User taps "Scanner mon QR"
- `welcome` вҶ’ `search`: User taps "Rechercher"
- `scanning` вҶ’ `confirm`: QR decoded, visit found
- `scanning` вҶ’ `welcome`: User taps "Annuler"
- `search` вҶ’ `confirm`: User selects visit result
- `search` вҶ’ `welcome`: User taps "Annuler"
- `confirm` вҶ’ `printing`: User taps "Confirmer" (API call succeeds)
- `confirm` вҶ’ `welcome`: User taps "Annuler"
- `printing` вҶ’ `success`: Print command succeeds
- `printing` вҶ’ `error`: Print command fails (with retry option)
- `printing` вҶ’ `welcome`: User cancels after error
- `success` вҶ’ `welcome`: Auto-reset after 8s, or user taps "Accueil"
- `scanning` вҶ’ `checkout-success`: Badge QR detected (check-out path)
- `checkout-success` вҶ’ `welcome`: Auto-reset after 5s
- `error` вҶ’ `welcome`: User taps "Retour Г  l'accueil"
- `error` вҶ’ `printing`: User taps "RГ©essayer"

**Idle timeout wiring:**
- `welcome`: 60s вҶ’ stays on welcome (already there)
- `scanning`: 30s вҶ’ reset to `welcome`
- `search`: 30s вҶ’ reset to `welcome`
- `confirm`: 30s вҶ’ reset to `welcome`
- `printing`: 30s from last event вҶ’ reset to `welcome` (only fires if printer never responds)
- `success`: 8s вҶ’ auto-advance to `welcome`
- `checkout-success`: 5s вҶ’ auto-advance to `welcome`
- `error`: No auto-timeout (user must tap to continue)

// apps/kiosk/lib/kiosk-api.ts

interface KioskConfig {

// Search visits by visitor name (for kiosk name search)

## Motion / Animation Patterns

Minimal animations (kiosk runs on low-power tablets/chromebooks):

| Element | Animation | Trigger | Duration |
|---------|-----------|---------|----------|
| Screen transitions | `opacity` + `translateY(20px)` вҶ’ `opacity-100 translateY(0)` | Phase change | 200ms |
| Success checkmark | `scale-in` (0.9 вҶ’ 1.0) | Check-in completes | 300ms |
| Printing indicator | `print-progress` pulsing indeterminate bar | Printing state | 2s loop |
| QR detection flash | Green `border` + `opacity` pulse | QR decoded | 400ms |
| Error flash | Red `border` + shake | Error state appears | 300ms |
| Button press | `active:scale-95` (CSS only) | Touch start | 100ms |
| Language toggle | `opacity` cross-fade | Toggle tap | 150ms |

**No auto-hide animations** (unlike Dashboard PTZ overlay). All kiosk content is intentionally visible at all times вҖ” no hover states or hidden controls.

**Respect `prefers-reduced-motion`:** All animations use `@media (prefers-reduced-motion: no-preference)` wrapper or `motion-safe:` Tailwind variant.

## Kiosk ID and Multi-Kiosk

Each kiosk instance sends `X-Kiosk-Id: {KIOSK_ID}` header with every API request. The API uses this to:
- Associate print jobs with the correct kiosk
- Track which kiosk handled which check-in (for audit logs)
- Allow per-kiosk printer configuration
