       - listen 80, server_name _
       - root /usr/share/nginx/html, index index.html
       - location /: try_files $uri $uri/ /index.html (SPA fallback)
       - location /_next/static/: expires 1y, Cache-Control public immutable
       - NO proxy_pass — SPA calls API directly from browser (D-16)

    3. docker/kiosk-entrypoint.sh:
       - #!/bin/sh
       - Start CUPS daemon: cupsd -f & (background)
       - sleep 2 (wait for CUPS ready)
       - If PRINTER_IP is set: lpadmin -p kiosk-printer -E -v "socket://${PRINTER_IP}:9100" -m raw -o printer-is-shared=false
       - cupsaccept kiosk-printer && cupsenable kiosk-printer
       - nginx -g "daemon off;" (foreground)
       - Make executable (chmod +x)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser (kiosk SPA) → Kiosk Docker container | nginx serves static content, no dynamic processing. No untrusted input. |
| Kiosk Docker container → CUPS → Network printer | ZPL data sent to network printer via socket://IP:9100. Printer is on LAN. |
| Kiosk Docker container → NestJS API | All data API calls go to NestJS with API key auth over HTTPS. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-KIO-01 | Spoofing | nginx static file serving | accept | nginx serves only pre-built static files on LAN. No dynamic content, no auth needed. |
| T-KIO-02 | Tampering | docker/kiosk-entrypoint.sh | mitigate | Entrypoint runs as root inside container. PRINTER_IP env var is read-only at startup. No user-submitted input reaches entrypoint. |
| T-KIO-03 | Information Disclosure | Static SPA files | accept | SPA contains no secrets. API_KEY is embedded at build time in JS bundle — acceptable for LAN kiosk (not internet-facing). |
| T-KIO-04 | Denial of Service | nginx serving | accept | Single kiosk on LAN. No external access. Container restarts automatically on failure. |
| T-KIO-SC | Tampering | npm/pip/cargo installs | mitigate | instascan@1.0.0 — slopcheck [OK] from RESEARCH.md. Verified in Package Legitimacy Audit. No [SUS] packages. |
</threat_model>
