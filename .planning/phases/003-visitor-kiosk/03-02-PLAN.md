    @Injectable() class KioskAuthGuard implements CanActivate:
      - Inject PrismaService
      - canActivate(context): Promise&lt;boolean&gt;
        - Get request from context.switchToHttp().getRequest()
        - Check X-API-Key header first:
          - If present: SHA-256 hash it, findFirst tenantApiKey where keyHash matches AND isActive is true
          - If found: set request.user = { orgId: keyRecord.organizationId, apiKeyId: keyRecord.id, scopes: keyRecord.scopes as string[] }
          - If not found: fall through to JWT (do NOT throw yet — the key could be invalid while a JWT is valid, but for kiosk routes we expect API key)
          - THROW UnauthorizedException("Clé API invalide") if key present but not found (don't fall through to JWT when API key is present and invalid)
        - If no API key header:
          - Check @Public() decorator via Reflector: if isPublic, return true
          - Fall through to JWT auth via passport: validate JWT bearer token
        - If neither API key nor JWT valid: throw UnauthorizedException
      - Register: @Injectable(), import Reflector from @nestjs/core
      - Use crypto from node:crypto for SHA-256

    The guard has three paths:
    1. X-API-Key present + valid → set request.user, return true
    2. X-API-Key present + invalid → throw 401 (no fallthrough to JWT)
    3. No X-API-Key → check @Public() skip, then JWT fallthrough

    DO NOT include Redis rate limiting (simplified for kiosk — rate limiting handled by global @fastify/rate-limit).
    DO NOT update lastUsedAt (simplified — kiosk API key usage tracking is not needed).
    DO NOT set X-RateLimit-* headers.

    Import: PrismaService from "../prisma/prisma.service"
    Import: crypto from "node:crypto"
    Import: Injectable, CanActivate, ExecutionContext, UnauthorizedException from "@nestjs/common"

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP (X-API-Key header) → KioskGuard → API | API key crosses network boundary into service |
| KioskService → child_process.exec('lp') | System command execution within container |
| KioskService → Prisma (visit data) | Database read for visit validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-KIO-05 | Spoofing | KioskAuthGuard (API key) | mitigate | SHA-256 hash prevents key exposure from DB leak. Nonce/timestamp not needed for LAN deployment. Key validation checks expiry + revocation. |
| T-KIO-06 | Tampering | KioskService.execAsync('lp') | mitigate | ZPL written to temp file (not passed as shell arg). ZPL escapeZpl removes ^ and ~ control chars. Command template is fixed string, no user input in command. |
| T-KIO-07 | Information Disclosure | /api/kiosk/ endpoints | mitigate | KioskAuthGuard with API key required on every endpoint. Visit data scoped to organization. No list-all-visits endpoint (only search by name). |
| T-KIO-08 | Denial of Service | POST /api/kiosk/print/:visitId | accept | Global @fastify/rate-limit protects /api/*. 15s timeout on lp command prevents hanging. |
| T-KIO-09 | Elevation of Privilege | KioskController reusing VisitorService | accept | Kiosk uses same checkIn/checkOut logic as dashboard. API key provides orgId scope. No additional privileges granted. |
</threat_model>
