## Spacing Scale

Both projects use Tailwind's default spacing scale (4px base unit). Phase 5 uses:

| Token | Value | Usage | Context |
|-------|-------|-------|---------|
| xs | 4px | Icon gaps, inline padding | Both |
| sm | 8px | Compact element spacing, badge padding | Both |
| md | 16px | Default element spacing, card padding | Both |
| lg | 24px | Section padding, card vertical gaps | Both |
| xl | 32px | Layout gaps between major sections, sidebar items | Both |
| 2xl | 48px | Page section breaks, modal padding | Both |
| 3xl | 64px | Page-level spacing, hero sections | Both |

Exceptions:
- **Sidebar navigation items**: 12px vertical padding (48px total height) — established in `admin-layout-shell.tsx` 
- **KPI cards min-height**: 120px — usage dashboard stat cards
- **Update banner**: 52px height for the vault-os dashboard notification bar

#### Accent (--marketing-primary) Reserved For:
- Primary CTA buttons (usage dashboard "Exporter", "Voir détails")
- KPI number values (total cameras, total clients)
- Active/selected navigation link in sidebar
- Link text ("Voir détails", "Voir historique")
- Focus ring on inputs
- Toggle/switch when active
- Loading spinner
- Documentation: accent highlights, link text, table of contents active indicator
- Support: chat widget accent color (if widget supports custom CSS variables)

| **Article routing** | If using Velite dynamic routes, articles at `/docs/guide-installation` etc. |

## Accessibility

- All interactive elements (buttons, links, inputs) must be focusable and have visible focus rings
- Focus ring color: `hsl(var(--marketing-ring))` for vault-app, `hsl(var(--shadcn-ring))` for vault-os
- Color contrast: All text must meet WCAG AA (4.5:1 for normal text, 3:1 for large text) against their backgrounds
- Chart data must include `aria-label` descriptions summarizing the visualized data
- Chart color encoding must not rely solely on color — include patterns or labels
- Update banner must be announced by screen readers on appear (`role="alert"` or `aria-live="polite"`)
- Documentation content must have proper heading hierarchy (h1 → h2 → h3)
- Code blocks must have `role="code"` and `aria-label` for the programming language
- Skip-to-content link for documentation pages
- All images in docs must have `alt` text
