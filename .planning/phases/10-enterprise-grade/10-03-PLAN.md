```typescript
@Injectable()
export class LicenseApiKeyGuard implements CanActivate {
  constructor(private prisma: PrismaService) {}
  async canActivate(context: ExecutionContext): Promise<boolean> {
    const apiKey = request.headers["x-api-key"];
    if (!apiKey) throw new UnauthorizedException("Clé API requise");
    const keyHash = crypto.createHash("sha256").update(apiKey).digest("hex");
    const keyRecord = await this.prisma.licenseApiKey.findFirst({ where: { keyHash, isActive: true } });
    if (!keyRecord) throw new UnauthorizedException("Clé API invalide");
    request.apiKeyInfo = { id: keyRecord.id, name: keyRecord.name };
    return true;
  }
}
```

**`webhook.processor.ts`** — Follow governance.processor.ts pattern exactly. Queue name: "webhook-delivery". The process() method:
1. Extract job data: `{ subscriptionId, eventType, payload, targetUrl, signingSecret, attemptNumber }`
2. Generate webhook ID (use job.id as unique delivery ID per D-13)
3. Sign payload via `this.webhookService.signPayload(payload, signingSecret)`
4. POST to targetUrl with headers: `Content-Type: application/json`, `X-Webhook-Id: job.id`, `X-Webhook-Signature: signedHeader`, `X-Webhook-Event: eventType`. Body: JSON.stringify(payload). Timeout: 30s (AbortSignal.timeout(30000)).
5. Log delivery result to DB (statusCode, responseBody[:10000]).
6. If response not OK and attemptNumber < 6: apply exponential backoff — delay from RETRY_SCHEDULE = [0, 60_000, 300_000, 900_000, 3_600_000, 86_400_000]. Use `job.moveToDelayed(Date.now() + RETRY_SCHEDULE[attemptNumber], job.token)`.
7. If attemptNumber >= 6 (exhausted): mark as permanently failed, log error. Emit EventEmitter2 event `"webhook.delivery-failed"` for gateway.
8. On success: emit `"webhook.delivery-completed"` event.

**`webhook.gateway.ts`** — Follow door.gateway.ts pattern exactly. Namespace: "/ws/webhooks". On connection: validate orgId from handshake auth, join org room. Listen for events:
- `@OnEvent("webhook.delivery-completed")` — emit to org room with delivery status
- `@OnEvent("webhook.delivery-failed")` — emit to org room with failure details
This enables the real-time delivery timeline in the dashboard (Plan 06).
