<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API key values | Raw API keys (osk_ prefix) must never appear in logs — only keyPrefix + "...XXX" |
| Webhook signing secrets | Stored encrypted at rest via pgp_sym_encrypt; never returned in API responses |
| IdP client secrets | OIDC clientSecret stored encrypted at rest; only used server-side during token exchange |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Information Disclosure | TenantApiKey.keyHash | mitigate | SHA-256 one-way hash — raw key never stored; only hash comparable on lookup |
| T-10-02 | Information Disclosure | WebhookSubscription.signingSecret | mitigate | Encrypt at rest via pgp_sym_encrypt (existing GovernanceService pattern); never include in API responses |
| T-10-03 | Information Disclosure | IdpConfig.clientSecret | mitigate | Encrypt at rest via pgp_sym_encrypt; never logged or returned in API responses |
| T-10-04 | Elevation of Privilege | TenantApiKey.scopes | mitigate | Scopes stored as JSON array; guard validates requested endpoint against scopes before execution (Plan 03) |
| T-10-05 | Tampering | Organization.logoUrl | mitigate | Validate logoUrl is a valid image URL or reject; prevent XSS via malformed URLs |
| T-10-SC | Tampering | npm installs (no new packages in this plan) | accept | No new npm packages installed in Plan 01 — all changes are schema/config/source only |
</threat_model>
