<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dashboard to API | All fetchWithAuth calls use JWT Bearer token — same auth boundary as existing dashboard |
| Login page to SSO redirect | Browser redirects to external IdP — must validate RelayState/state parameter on return |
| Logo URL input | User-supplied URL rendered in dashboard — must prevent XSS via Content-Security-Policy |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-26 | Information Disclosure | API key display in dashboard | mitigate | Raw key shown ONLY on creation in Dialog; closed dialog cannot be reopened; keyHash never sent to client |
| T-10-27 | Information Disclosure | Webhook signing secret in dashboard | mitigate | Signing secret NEVER displayed after creation; only masked version (first 8 + last 4 chars) |
| T-10-28 | Cross-Site Scripting | Organization.logoUrl rendering | mitigate | Render logo as img tag with CSP img-src directive; sanitize URL before rendering |
| T-10-29 | Cross-Site Scripting | Organization.displayName | mitigate | React auto-escapes JSX content; rendered as textContent, not dangerouslySetInnerHTML |
| T-10-30 | Elevation of Privilege | Dashboard admin pages | mitigate | All enterprise pages gated by minRole: ADMIN in nav-config; server-side @Roles("ADMIN") |
| T-10-SC | Tampering | npm installs (no new packages) | accept | No new npm packages — all components use existing shadcn/ui, lucide-react, and repo/design |
</threat_model>
