## Color

| Role | Value (HSL) | Usage |
|------|-------------|-------|
| Dominant (60%) | `228 20% 4%` | Page background, main surface |
| Secondary (30%) | `228 24% 6%` (card) / `228 16% 12%` (muted) | Cards, sidebar, navigation, dialogs, dropdowns, settings panels |
| Accent (10%) | `190 90% 50%` (cyan) | Primary buttons, links, focus rings, selected states, glow effects, progress bars, active nav items |
| Destructive | `0 84% 60%` (red) | Destructive actions only (revoke key, delete webhook, disable SSO) |
| Warning | `35 92% 50%` (amber) | License expiry warnings, webhook delivery failures, SSO config validation warnings, compliance report generation alerts |
| Success | `160 84% 39%` (green) | SSO test success, webhook delivery success, API key creation confirmation, license activation confirmation |

**Accent reserved for:** primary buttons (variant `default`), hyperlinks, `focus-visible` rings (`--shadcn-ring`), selected/active states in lists and tables, progress bar fill, API usage sparklines, navigation item active indicators. **Never:** body text, card backgrounds, destructive buttons, warning badges.

**Light mode:** `[data-theme="light"]` variant swaps dominant to `0 0% 100%` (white), secondary to `220 20% 97%`, accent to `190 90% 40%`, destructive to `0 84% 55%`. All components must render correctly in both modes.

> **Source:** Existing design system from `globals.css` (v2.0 Phase 6). No changes for Phase 10.

### Phase-Specific Copy Notes

- **SSO login page:** When IdP is configured, the login page displays "Sign in with SSO" as the primary button (accent variant), with a secondary "Sign in with email" link below. When no IdP is configured, the SSO section is hidden.
- **API key creation:** On creation, the full key is displayed once with a copy-to-clipboard button and the message: "Copy your API key now. You will not be able to see it again." After dismissal, only the key prefix (`osk_abc123...`) is shown.
- **Webhook signing secret:** Never displayed in UI after creation. Only a masked version (`whsec_abc...xyz`) shown in the subscription details. Rotation generates a new secret; the old secret is invalidated immediately.
- **Compliance report generation:** Button shows a loading spinner with "Generating report..." during server-side PDF generation. On completion, the PDF downloads automatically via blob URL.
- **i18n:** All copy is provided in English (primary development language for Phase 10 settings/admin pages). French translations follow the existing `apps/dashboard/lib/i18n/` dictionary pattern.

> **Source:** User confirmed primary CTA and SSO copy. Remaining values use enterprise-SaaS defaults per user instruction.
